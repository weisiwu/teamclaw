## 经验教训

### 1. 循环依赖检测
- 定期检查 `import from "./xxx"` 是否指向自身
- 使用 TypeScript 编译检测循环依赖

### 2. 安全意识
- 任何用户输入都应参数化，避免字符串拼接
- 移除开发环境特殊认证逻辑

### 3. Mock 数据管理
- 开发阶段使用 Mock 数据应有明确标记
- 生产环境应完全移除 Mock

### 4. 文档同步
- 代码变更后应及时更新文档
- 导入流程等复杂逻辑应有完整步骤说明
