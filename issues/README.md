# TeamClaw 迭代回顾（Issues）

> 本目录存放**每轮迭代结束后的回顾总结**，用于复盘和知识沉淀。

## 定位说明

```
docs/iterators/processing/  →  前瞻性任务规格书（"要做什么"）
docs/iterators/done/        →  已完成任务归档（"做了什么"）
issues/                     →  迭代回顾总结（"做得怎么样、学到了什么"）
```

**`issues/` ≠ 任务列表**。任务管理在 `docs/iterators/` 中进行，`issues/` 只在一轮迭代结束后生成回顾报告。

## 经验教训（跨迭代沉淀）

### 1. 循环依赖检测
- 定期检查 `import from "./xxx"` 是否指向自身
- 使用 TypeScript 编译检测循环依赖

### 2. 安全意识
- 任何用户输入都应参数化，避免字符串拼接
- 移除开发环境特殊认证逻辑

### 3. Mock 数据管理
- 开发阶段使用 Mock 数据应有明确标记
- 生产环境应完全移除 Mock

### 4. 文档同步
- 代码变更后应及时更新文档
- 导入流程等复杂逻辑应有完整步骤说明

### 5. 跨项目隔离
- 不同 app 的迭代任务不要混放在同一目录
- 每个 app 维护独立的 `docs/iterators/` 目录
