## 1. agents.ts — Agent 团队编排

**文件路径**: `server/src/constants/agents.ts`

### Agent 等级

| 等级 | 标签   | 说明                    |
| ---- | ------ | ----------------------- |
| Lv1  | 执行层 | 程序员（coder1/coder2） |
| Lv2  | 策划层 | PM / 审查者             |
| Lv3  | 决策层 | 主管（main）            |

### Agent 团队

| Agent      | 角色      | 等级 | 群聊可见 |
| ---------- | --------- | ---- | -------- |
| `main`     | 主管      | Lv3  | ✅       |
| `pm`       | 产品经理  | Lv2  | ✅       |
| `reviewer` | 代码审查  | Lv2  | ❌       |
| `coder1`   | 程序员1号 | Lv1  | ❌       |
| `coder2`   | 程序员2号 | Lv1  | ❌       |

### 派发权限矩阵

```
main     → pm, reviewer, coder1, coder2
pm       → coder1, coder2
reviewer → coder1, coder2
```

### 核心辅助函数

| 函数                          | 说明                   |
| ----------------------------- | ---------------------- |
| `canDispatch(from, to)`       | 判断是否有权指派       |
| `isReverseDispatch(from, to)` | 判断是否反向指派       |
| `getAgentByName(name)`        | 按名称获取 Agent 配置  |
| `getAgentsByLevel(level)`     | 获取指定等级所有 Agent |
| `getSubordinates(name)`       | 获取下级 Agent         |
| `getTeamOverview()`           | 获取团队全貌           |

## 2. roles.ts — 角色与权限

**文件路径**: `server/src/constants/roles.ts`

### 角色与权重

| 角色标识     | 标签     | 权重 |
| ------------ | -------- | ---- |
| `admin`      | 管理员   | 10   |
| `vice_admin` | 副管理员 | 7    |
| `member`     | 普通员工 | 3    |

### Agent 访问矩阵

| 角色         | @main | @pm | @coder | @reviewer |
| ------------ | ----- | --- | ------ | --------- |
| `admin`      | ✅    | ✅  | ✅     | ✅        |
| `vice_admin` | ✅    | ✅  | ❌     | ❌        |
| `member`     | ✅    | ❌  | ❌     | ❌        |

### 核心辅助函数

| 函数                          | 说明                       |
| ----------------------------- | -------------------------- |
| `canAccessAgent(role, agent)` | 判断是否有权访问 Agent     |
| `getPmCapability(role)`       | 获取与 pm 交互时的能力范围 |
| `compareRoleWeight(a, b)`     | 比较两个角色的权重差异     |
| `getHigherRole(role)`         | 获取更高优先级的角色       |

## 3. apiErrorCodes.ts — API 错误码

**文件路径**: `server/src/constants/apiErrorCodes.ts`

### 错误码分段

| 段   | 范围       | 含义                 |
| ---- | ---------- | -------------------- |
| 1xxx | 通用错误   | 认证、授权、参数校验 |
| 2xxx | 资源操作   | CRUD 相关            |
| 3xxx | 业务逻辑   | 状态、转换、冲突     |
| 4xxx | 第三方服务 | Git、构建、LLM       |
| 5xxx | 系统错误   | 数据库、缓存等       |

### 核心函数

```typescript
import { API_ERROR_CODES, getErrorMessage } from '../constants/apiErrorCodes.js';

getErrorMessage(API_ERROR_CODES.UNAUTHORIZED);
// → '未认证，请登录后操作'
```
