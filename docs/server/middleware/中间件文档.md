## 1. auth.ts — 身份认证与权限中间件

**文件路径**: `server/src/middleware/auth.ts`

**依赖**: `server/src/utils/jwt.js`、`server/src/services/permissionService.js`

#### 职责

从 HTTP 请求的 `Authorization: Bearer <token>` Header 中提取并验证 JWT Token，提取用户身份（userId + role），并可选地对指定 Agent 执行权限检查。所有中间件均扩展 Express 的 `Request` 类型，在 `req.user` 中附加用户信息。

### 1.1 核心类型

```typescript
export interface AuthRequest extends Request {
  user?: {
    id: string;
    role: Role;
  };
  permission?: PermissionCheckResult;
}

type AuthRequestHandler = (req: AuthRequest, res: Response, next: NextFunction) => void;
```

### 1.2 Token 提取机制

```typescript
function extractUserFromToken(req: AuthRequest): JwtPayload | null;
function extractUser(req: AuthRequest): { id: string; role: Role } | null;
```

| 函数                   | 说明                                            |
| ---------------------- | ----------------------------------------------- |
| `extractUserFromToken` | 从 `Authorization: Bearer <token>` 解析 JWT     |
| `extractUser`          | 组合提取用户身份，返回 `{ id, role }` 或 `null` |

**验证流程**:

1. 检查 `Authorization` Header 是否以 `Bearer ` 开头
2. 提取 token 字符串（去掉 `Bearer ` 前缀）
3. 调用 `verifyToken()` 验证签名和有效期
4. 验证失败返回 `null`

### 1.3 导出中间件

| 中间件                     | 说明                                           | 适用场景                      |
| -------------------------- | ---------------------------------------------- | ----------------------------- |
| `requireAuth`              | 要求有效身份，不限 Agent                       | 需确认身份但不限权限的接口    |
| `requirePermission(agent)` | 验证对特定 Agent 的权限                        | 限制只能调用某个 Agent 的接口 |
| `optionalAuth`             | 可选身份，有 Token 时自动解析附加到 `req.user` | 公开接口但支持已登录用户      |
| `requireAdmin`             | 要求 admin 或 vice_admin 角色                  | 管理后台接口                  |

#### `requireAuth`

```typescript
export function requireAuth(req: AuthRequest, res: Response, next: NextFunction): void;
```

**行为**:

| 场景                   | 响应                                           |
| ---------------------- | ---------------------------------------------- |
| 无 Token 或 Token 无效 | `401 未提供有效身份信息，请通过登录获取 Token` |
| 验证通过               | `req.user = { id, role }`，`next()`            |

#### `requirePermission(agent: AgentName)`

```typescript
export function requirePermission(agent: AgentName): AuthRequestHandler;
```

**行为**:

| 场景            | 响应                                           |
| --------------- | ---------------------------------------------- |
| 无有效身份      | `401 未提供有效身份信息`                       |
| 无该 Agent 权限 | `403 没有权限`                                 |
| 验证通过        | `req.user` + `req.permission` 均填充，`next()` |

#### `requireAdmin`

```typescript
export function requireAdmin(req: AuthRequest, res: Response, next: NextFunction): void;
```

**行为**:

| 场景                | 响应                                |
| ------------------- | ----------------------------------- |
| 未登录              | `401 未提供有效身份信息`            |
| 非 admin/vice_admin | `403 需要管理员或副管理员权限`      |
| 验证通过            | `req.user = { id, role }`，`next()` |

#### `optionalAuth`

```typescript
export function optionalAuth(req: AuthRequest, _res: Response, next: NextFunction): void;
```

**行为**: 不强制要求身份信息，如有有效 Token 则附加到 `req.user`，无论是否有 Token 均 `next()`。

### 1.4 Agent 权限矩阵

| 用户角色   | @main | @pm | @coder | @reviewer |
| ---------- | ----- | --- | ------ | --------- |
| admin      | ✅    | ✅  | ✅     | ✅        |
| vice_admin | ✅    | ✅  | ❌     | ❌        |
| member     | ✅    | ❌  | ❌     | ❌        |

**权限检查实现**: 调用 `checkPermission(user.role, agent)`（来自 `permissionService`）。

## 2. errorHandler.ts — 统一错误处理中间件

**文件路径**: `server/src/middleware/errorHandler.ts`

**依赖**: `server/src/utils/response.js`

#### 职责

提供全局统一的错误处理、404 兜底和异步函数自动捕获。必须注册在所有路由之后，作为 Express 错误处理中间件（4 参数）。

### 核心导出

| 导出                  | 说明                                              |
| --------------------- | ------------------------------------------------- |
| `unifiedErrorHandler` | 全局错误处理中间件（必须 4 参数）                 |
| `notFoundHandler`     | 404 兜底，处理所有未匹配路由                      |
| `asyncHandler`        | 异步路由包装器，自动捕获 Promise reject 并传 next |
| `ApiError`            | 可抛出的自定义错误类                              |
| `ErrorCodes`          | 错误码常量对象                                    |

### 统一错误响应格式

```json
{
  "success": false,
  "code": 500,
  "errorCode": "INTERNAL_ERROR",
  "message": "Internal server error",
  "requestId": "uuid",
  "timestamp": "ISO8601",
  "path": "/api/v1/xxx",
  "method": "GET"
}
```

### 2.1 核心类型

```typescript
export class ApiError extends Error {
  public statusCode: number;
  public errorCode: string;
  public requestId: string;

  constructor(
    statusCode: number = 500,
    errorCode: string = 'INTERNAL_ERROR',
    message: string = 'Internal server error'
  );
}

export const ErrorCodes = {
  BAD_REQUEST: 'BAD_REQUEST',
  VALIDATION_ERROR: 'VALIDATION_ERROR',
  UNAUTHORIZED: 'UNAUTHORIZED',
  FORBIDDEN: 'FORBIDDEN',
  NOT_FOUND: 'NOT_FOUND',
  RATE_LIMITED: 'RATE_LIMITED',
  INTERNAL_ERROR: 'INTERNAL_ERROR',
  DATABASE_ERROR: 'DATABASE_ERROR',
  EXTERNAL_SERVICE_ERROR: 'EXTERNAL_SERVICE_ERROR',
} as const;
```

### 2.2 错误自动推断逻辑

`unifiedErrorHandler` 根据错误类型自动推断 HTTP 状态码：

| 错误类型              | 推断逻辑                     | statusCode       |
| --------------------- | ---------------------------- | ---------------- |
| `ApiError` 实例       | 使用实例的 `statusCode` 属性 | `err.statusCode` |
| JSON 解析错误         | `err.name === 'SyntaxError'` | 400              |
| 消息含 `not found`    | 正则匹配 `err.message`       | 404              |
| 消息含 `unauthorized` | 正则匹配 `err.message`       | 401              |
| 消息含 `forbidden`    | 正则匹配 `err.message`       | 403              |
| 其他未分类错误        | 默认                         | 500              |

**开发环境行为**: 额外打印 `err.name`、`err.message` 和完整 `err.stack`。

### asyncHandler 使用示例

```typescript
import { asyncHandler, success } from '../middleware/errorHandler.js';
import { getUsers } from '../services/userService.js';

router.get(
  '/users',
  asyncHandler(async (req, res, next) => {
    const users = await getUsers();
    res.json(success(users));
  })
);
```

## 3. projectAccess.ts — 项目权限检查中间件

**文件路径**: `server/src/middleware/projectAccess.ts`

**依赖**: `server/src/middleware/auth.js`（需先执行 `requireAuth`）、`server/src/db/sqlite.js`

#### 职责

检查当前用户是否有权操作指定版本（基于版本创建者或角色）。**安全修复（iter-03）**: 身份信息强制从 JWT 验证后的 `req.user` 获取，彻底废弃从 HTTP Header 读取用户身份的做法，杜绝身份伪造漏洞。

### 权限规则

| 条件                             | 结果                             |
| -------------------------------- | -------------------------------- |
| admin / vice_admin               | ✅ 直接放行                      |
| 版本的 `created_by` 等于当前用户 | ✅ 版本创建者本人                |
| 其他                             | ❌ HTTP 403 `没有权限操作此版本` |

### 中间件

```typescript
export function requireProjectAccess(req: AuthRequest, res: Response, next: NextFunction): void;
```

**前置条件**: 调用方应先执行 `requireAuth` 中间件，确保 `req.user` 已填充。

**参数来源**: `req.params.id`（版本 ID，UUID 格式）

**查询语句**:

```sql
SELECT id, created_by, project_id FROM versions WHERE id = ?
```

**完整响应**:

| 场景                        | 响应                                                   |
| --------------------------- | ------------------------------------------------------ |
| 未登录（`req.user` 不存在） | `401 未提供有效身份信息`                               |
| 版本不存在                  | `404 版本不存在`                                       |
| admin/vice_admin            | `next()`                                               |
| 版本创建者                  | `next()`                                               |
| 无权限                      | `403 没有权限操作此版本，仅版本创建者或管理员可以操作` |
| 数据库异常                  | `500 权限检查失败`                                     |

## 4. validation.ts — API 输入校验中间件

**文件路径**: `server/src/middleware/validation.ts`

**依赖**: `zod`

#### 职责

使用 [Zod](https://zod.dev) 对请求参数（路径参数、查询参数、请求体）进行运行时类型校验，统一返回 400 错误。校验通过后自动用 Zod 解析后的干净数据（含默认值）替换原始请求体。

### 4.1 内置 Schema

| Schema 名称           | 用途           | 校验规则                                                       |
| --------------------- | -------------- | -------------------------------------------------------------- |
| `versionIdSchema`     | 版本 ID        | 以 `v` 开头，后跟数字/下划线/字母                              |
| `uuidSchema`          | UUID 格式      | 标准 UUID v4 格式                                              |
| `paginationSchema`    | 分页参数       | `page` ≥ 1，`pageSize` 1-100，默认 `{ page: 1, pageSize: 10 }` |
| `versionStatusSchema` | 版本状态       | `draft` \| `published` \| `archived`                           |
| `buildStatusSchema`   | 构建状态       | `pending` \| `building` \| `success` \| `failed`               |
| `semverSchema`        | 语义化版本号   | 标准 semver 格式（如 `1.0.0-alpha.1`）                         |
| `createVersionSchema` | 创建版本请求体 | `version`(必填，semver) + `title`(必填) + 可选字段             |
| `updateVersionSchema` | 更新版本请求体 | 所有字段可选                                                   |
| `rollbackSchema`      | 回滚请求体     | `target`(必填) + 可选 `type`/`createBranch`                    |
| `buildConfigSchema`   | 构建配置请求体 | 可选 `buildCommand`/`projectPath`                              |

### 4.2 中间件工厂函数

| 工厂函数                | 校验目标        | 校验失败响应                                                    |
| ----------------------- | --------------- | --------------------------------------------------------------- |
| `validateId(schema)`    | `req.params.id` | `{ code: 400, message: '参数校验失败: {具体错误}' }`            |
| `validateBody(schema)`  | `req.body`      | `{ code: 400, message: '请求体验证失败: {字段}: {错误}, ...' }` |
| `validateQuery(schema)` | `req.query`     | `{ code: 400, message: '查询参数校验失败: {具体错误}' }`        |

**使用示例**:

```typescript
import {
  validateId,
  validateBody,
  validateQuery,
  uuidSchema,
  createVersionSchema,
  paginationSchema,
} from '../middleware/validation.js';

// 路径参数校验
router.get('/versions/:id', validateId(uuidSchema), (req, res) => { ... });

// 请求体校验
router.post('/versions', validateBody(createVersionSchema), (req, res) => { ... });

// 查询参数校验
router.get('/versions', validateQuery(paginationSchema), (req, res) => { ... });
```

## 5. 安全配置 — Helmet / CORS / Rate Limiting

> 以下中间件配置均在 `server/src/index.ts` 中内联设置，未单独抽取为文件。

### 5.1 Helmet 安全头

**依赖**: `helmet`

```typescript
app.use(helmet({ ... }))
```

**配置项**:

| 配置                           | 值                                | 说明                         |
| ------------------------------ | --------------------------------- | ---------------------------- |
| `contentSecurityPolicy`        | 严格 CSP 指令                     | 防止 XSS/注入                |
| `xContentTypeOptions`          | `true`                            | 禁止 MIME 类型嗅探           |
| `xFrameOptions`                | `{ action: 'deny' }`              | 防止点击劫持                 |
| `xXssProtection`               | `true`                            | 启用浏览器 XSS 过滤器        |
| `referrerPolicy`               | `strict-origin-when-cross-origin` | Referrer 策略                |
| `hsts`                         | 1年 + includeSubDomains + preload | 强制 HTTPS                   |
| `dnsPrefetchControl`           | `{ allow: false }`                | 禁止 DNS 预取                |
| `permittedCrossDomainPolicies` | `none`                            | 禁止 Flash cross-domain 策略 |
| `crossOriginEmbedderPolicy`    | `false`                           | 允许嵌入跨域资源             |

### 5.2 CORS 跨域配置

**依赖**: `cors`

```typescript
const allowedOrigins = (process.env.ALLOWED_ORIGINS || 'http://localhost:3000')
  .split(',')
  .map(s => s.trim());

app.use(cors({
  origin: (origin, callback) => { ... },
  credentials: true,
  methods: ['GET', 'POST', 'PUT', 'DELETE', 'PATCH', 'OPTIONS'],
  allowedHeaders: ['Content-Type', 'Authorization', 'X-Requested-With', 'X-Request-Id'],
  exposedHeaders: ['X-Request-Id', 'Content-Disposition'],
}))
```

**origin 校验逻辑**:

| 来源                          | 结果    |
| ----------------------------- | ------- |
| 无 origin（如移动端/curl）    | ✅ 允许 |
| 在 `ALLOWED_ORIGINS` 白名单中 | ✅ 允许 |
| 开发环境 + 含 `localhost`     | ✅ 允许 |
| 其他                          | ❌ 拒绝 |

**环境变量**:

| 变量名            | 说明                     | 默认值                  |
| ----------------- | ------------------------ | ----------------------- |
| `ALLOWED_ORIGINS` | 允许的域，多个用逗号分隔 | `http://localhost:3000` |

### 5.3 Rate Limiting 限流

**依赖**: `express-rate-limit`

#### 全局限流（`defaultLimiter`）

| 配置项   | 值                                                   | 说明 |
| -------- | ---------------------------------------------------- | ---- |
| 触发路径 | 所有 `/api/v1/*` 路由                                |      |
| 时间窗口 | 60 秒                                                |      |
| 请求上限 | 100 请求/窗口                                        |      |
| 响应码   | `429`                                                |      |
| 响应体   | `{ code: 429, message: '请求过于频繁，请稍后再试' }` |

#### 严格限流（`strictLimiter`）

| 配置项   | 值                                    | 说明         |
| -------- | ------------------------------------- | ------------ |
| 触发路径 | `/api/v1/auth/login`、`/api/v1/users` | 敏感接口     |
| 时间窗口 | 60 秒                                 |              |
| 请求上限 | 10 请求/窗口                          | 防止暴力破解 |
| 响应码   | `429`                                 |              |

**RateLimit 标准响应头**（`standardHeaders: true`）:

| Header                | 说明                   |
| --------------------- | ---------------------- |
| `RateLimit-Limit`     | 窗口内最大请求数       |
| `RateLimit-Remaining` | 窗口内剩余请求数       |
| `RateLimit-Reset`     | 窗口重置时间戳（Unix） |

## 中间件注册顺序

```typescript
// 1. 安全头（CORS 在路由之前）
app.use(cors({ ... }));
app.use(helmet({ ... }));

// 2. Body 解析
app.use(express.json());

// 3. 限流（在路由之前）
app.use('/api/v1', defaultLimiter);
app.use('/api/v1/auth/login', strictLimiter);
app.use('/api/v1/users', strictLimiter);

// 4. 业务路由
app.use('/api/v1', routes);

// 5. 404 兜底
app.use(notFoundHandler);

// 6. 全局错误处理（最后）
app.use(unifiedErrorHandler);
```
