# P13 - 导入项目

**路由**: `/import`  
**文件**: `app/import/page.tsx`  
**类型**: 客户端页面（`'use client'`）

---

## 功能概述

通过 Git URL 导入项目，自动检测技术栈和配置。

## 页面结构

```
┌─────────────────────────────────────────────────────┐
│ Header: 导入项目                                    │
├─────────────────────────────────────────────────────┤
│ Git 地址                                            │
│ [输入 Git URL (https://github.com/xxx/yyy)]        │
│                                                     │
│ 认证 (可选)                                         │
│ 用户名: [输入用户名]                                │
│ 密码/Token: [输入密码或 Personal Access Token]     │
│                    [检测] [导入]                   │
├─────────────────────────────────────────────────────┤
│ 导入进度                                            │
│ [=====>          ] 60%                            │
│ ✓ 克隆代码库                                       │
│ ○ 检测技术栈                                        │
│ ○ 分析项目结构                                      │
│ ○ 创建项目记录                                      │
└─────────────────────────────────────────────────────┘
```

## 导入流程 (8 步)

| 步骤 | 说明 |
|---|---|
| 1 | 克隆代码库 |
| 2 | 扫描文件结构 |
| 3 | 检测技术栈 |
| 4 | 分析依赖配置 |
| 5 | 检测打包机制 |
| 6 | 生成功能定位 |
| 7 | 分析历史改动 |
| 8 | 完成导入 |

## API 调用

| 端点 | 方法 | 说明 |
|---|---|---|
| `POST /api/v1/projects/import` | 导入 | 触发项目导入 |
| `GET /api/v1/projects/import/:taskId/status` | 状态 | 获取导入进度 |

## 相关文件

- `app/import/page.tsx` — 本页
- `components/import/` — 导入组件
- `lib/api/projects.ts` — 项目 API
- `server/src/routes/project.ts` — 后端路由
- `server/src/services/importOrchestrator.ts` — 导入编排器
