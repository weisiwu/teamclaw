# 消息队列

## 页面路由

```
/messages
```

## 页面功能描述

消息队列（Message Queue）管理页面，实时展示消息处理状态：
- **队列统计**：队列总数、处理中数量、排队中数量
- **当前处理中消息**：高优先级卡片展示
- **排队消息列表**：所有待处理消息，含抢占按钮
- **测试消息发送**：手动发送测试消息，触发优先级调度
- **抢占规则说明**：展示优先级计算规则

消息队列每 5 秒自动刷新。

## 优先级计算规则

```
优先级 = 角色权重 × 消息紧急度
```

| 角色 | 权重 |
|------|------|
| 管理员 | 10 |
| 副管理员 | 7 |
| 员工 | 3 |

**抢占条件**：新消息优先级 > 当前任务优先级 × 1.5

## 优先级示例

| 消息 | 计算 | 优先级 |
|------|------|--------|
| 管理员 + 紧急消息 | 10×3 | P30 |
| 副管理员 + 高优先级 | 7×2 | P15 |
| 管理员 + 普通消息 | 10×1 | P10 |
| 员工 + 普通消息 | 3×1 | P3 |

## 消息合并

同一用户 5 分钟内的连续消息会自动合并为一条。
