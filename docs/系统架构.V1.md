## 1. 整体框架

### 1.1 核心定位

TeamClaw 是一个**业务接入框架**，核心能力：
- 业务代码导入 → 自动解析 → 生成记忆
- 群聊消息驱动 → 任务调度 → Agent 协作执行
- 版本管理 → 构建交付 → 后台可视化

### 1.2 系统组成

```
┌─────────────────────────────────────────────────────────┐
│                    Web 后台管理平台                       │
│  职责：项目导入、任务管理、版本管理、配置、监控              │
└────────────────────┬────────────────────────────────────┘
                     │ 调用
                     ▼
┌─────────────────────────────────────────────────────────┐
│                   三层架构核心                            │
├─────────────────────────────────────────────────────────┤
│  Layer 1: WorkBuddy（入口层）                            │
│  核心职责：                                               │
│  1. 数据转换：微信/飞书消息 → 统一任务格式                  │
│  2. 角色暴露：将 Layer3 的 main、pm 暴露到群聊             │
│  3. 身份鉴权、消息队列、优先级排序                          │
│  模型：轻量模型（DeepSeek-Chat / Kimi）                   │
├─────────────────────────────────────────────────────────┤
│  Layer 2: MiddlewareClaw（调度层）                       │
│  核心职责：                                               │
│  1. 任务队列管理：统一管理所有任务的生命周期                 │
│  2. 记忆优化：每条任务后续都进行记忆化处理                   │
│  3. 上下文补充：从项目记忆中召回相关上下文                   │
│  4. 任务拆解、业务调度、Prompt 生成                        │
│  模型：中等模型（GPT-4o-mini / DeepSeek-V3）              │
├─────────────────────────────────────────────────────────┤
│  Layer 3: Agents OpenClaw（执行层）                      │
│  核心职责：                                               │
│  1. 静默执行：接收 Layer2 的精炼 Prompt，安静地拆分任务     │
│  2. 具体实现：代码编写、文件操作、Git 操作、测试             │
│  3. 结果上报：将执行结果返回 Layer2                        │
│  模型：强模型（Claude Sonnet / GPT-4o）                   │
│  角色：main、pm（暴露在群）、coder1、coder2、reviewer（隐藏）│
└─────────────────────────────────────────────────────────┘
```

### 1.3 数据流

```
用户群聊消息（微信/飞书）
    │
    ▼
Layer1 (WorkBuddy)
    ├── 数据转换：消息 → 任务格式
    ├── 角色暴露：用户感知到 main、pm 的存在
    ├── 身份鉴权：判断用户角色和权限
    └── 任务生成 → tasks/inbox/{task_id}_{priority}.json
    │
    ▼
Layer2 (MiddlewareClaw)
    ├── 任务队列管理：入队、优先级排序、抢占
    ├── 记忆优化：召回项目记忆、压缩上下文
    ├── 上下文补充：RAG 检索相关代码/文档
    ├── 任务拆解：大任务 → 子任务
    └── Prompt 生成 → tasks/dispatched/{task_id}.json
    │
    ▼
Layer3 (Agents OpenClaw)
    ├── main：任务分配、质量把控
    ├── pm：需求细化、群消息收集
    ├── coder1/coder2：代码编写
    ├── reviewer：代码审查
    └── 执行结果 → tasks/completed/{task_id}_result.json
    │
    ▼
Layer2 汇总结果 → Layer1 回传群聊
```

### 1.5 关键设计原则

1. **角色暴露机制**：只有 main 和 pm 在群聊中可见，避免用户直接给低级 Agent 下达混乱指令
2. **任务队列管理**：Layer2 统一管理任务队列，支持优先级、抢占、挂起
3. **记忆优化**：每个任务完成后都要记忆化，方便后续检索和复用
4. **上下文补充**：通过 RAG 从项目记忆中召回相关信息，减少重复询问
