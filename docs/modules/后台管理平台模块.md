## 1. 定位

Web 端可视化配置与监控面板，是系统的管理入口。

---

## 2. 功能模块

| 模块 | 功能项 |
|------|--------|
| **初始化** | 项目导入向导 |
| **任务管理** | 任务列表、任务详情、完成、取消 |
| **版本管理** | 版本列表、详情、创建分支、指定主分支、构建、重新构建、下载产物 |
| **辅助能力管理** | 能力列表、能力开关 |
| **定时任务管理** | 查看/添加定时任务（仅 Prompt 驱动） |
| **项目文档管理** | 在线浏览所有文档 |
| **Token 消费** | 按天/周/月统计 Token 用量与成本 |

## 11. API 定义（本模块新增）

> 后台管理平台大部分页面复用其他模块的 API（项目导入、任务、版本、辅助能力、人员管理等）。以下仅列出本模块独有的 API。

| 方法 | 路径 | 说明 |
|------|------|------|
| `GET` | `/api/v1/cron-jobs` | 获取定时任务列表 |
| `POST` | `/api/v1/cron-jobs` | 添加定时任务 |
| `PUT` | `/api/v1/cron-jobs/:jobId` | 编辑定时任务 |
| `DELETE` | `/api/v1/cron-jobs/:jobId` | 删除定时任务 |
| `PUT` | `/api/v1/cron-jobs/:jobId/toggle` | 启动/停止定时任务 |
| `GET` | `/api/v1/token-stats` | 获取 Token 消费统计（支持按天/周/月、按层级/任务维度） |
| `GET` | `/api/v1/token-stats/trend` | 获取 Token 消费趋势数据（折线图） |
| `GET` | `/api/v1/dashboard/overview` | 获取仪表盘概览数据（项目/任务/版本/Token 汇总） |

**POST /api/v1/cron-jobs 请求示例**

```json
{
  "name": "每天 2:00 刷新项目记忆",
  "cron": "0 2 * * *",
  "prompt": "请刷新项目记忆，重新解析代码和文档"
}
```

**GET /api/v1/token-stats 响应示例**

```json
{
  "code": 0,
  "data": {
    "period": "week",
    "totalTokens": 125000,
```
