## 1. 任务生命周期

```
消息触发 → 对话确认 → 任务创建 → 进行中 → 测试中 → 已完成
                                    │                    │
                                    └──── 已取消 ◄────────┘
```

---

## 2. 任务流转详细流程

| 阶段 | 触发条件 | 行为 |
|------|---------|------|
| **触发** | 用户 @main | main 开始与用户对话讨论需求 |
| **确认** | 管理员明确同意 或 main 主动确认获得肯定答复 | 需求锁定 |
| **创建** | 确认后 | pm 在群内同步任务信息 + 生成任务详情链接 |
| **进行中** | 任务委派给 coder | 自动进入开发状态 |
| **测试中** | coder 完成开发 | main 群内通知「任务已完成」 |
| **已完成** | 在任务详情页点击「完成」 | 归档 + 自动版本升级 |
| **已取消** | 在任务详情页点击「取消」 | 归档 |


## 4. 任务持久化存储

### 4.1 存储位置

```
~/.openclaw/tasks/
  ├── inbox/         # 待处理任务
  ├── dispatched/    # 已分发任务
  ├── completed/     # 已完成任务
  ├── suspended/     # 被挂起任务
  └── archived/      # 归档任务
```

### 4.2 任务文件命名

```
tasks/inbox/{task_id}_{priority}.json
tasks/dispatched/{task_id}.json
tasks/completed/{task_id}_result.json
```

---

## 5. 任务记忆向量化

每个任务完成后都要进行记忆化处理：

1. **生成任务摘要**：使用中等模型压缩任务对话和改动
2. **向量化存储**：将摘要 Embedding 后存入向量数据库
3. **建立索引**：摘要 → 原始任务文件的映射
4. **支持检索**：后续可通过语义搜索召回相似任务

---

## 6. 任务详情页

任务详情页可随时查看：

- **基本信息**：任务 ID、标题、状态、优先级
- **时间信息**：创建时间、完成时间、耗时
- **改动摘要**：修改了哪些文件，做了什么改动
- **关联 commit**：Git commit 哈希和消息
- **参与 Agent**：哪些 Agent 参与了任务
- **Token 消耗**：任务消耗的 Token 数量
- **操作按钮**：完成、取消

## 7. 任务创建流程详解

```
用户 @main "我想在首页加个按钮"
    │
    ▼
main 开始对话
    main: "好的，请问按钮的文字是什么？"
    用户: "立即购买"
    main: "按钮点击后要跳转到哪里？"
    用户: "商品详情页"
    main: "明白了，我来安排开发"
    │
    ▼
main 确认需求（等待用户明确同意）
    main: "确认一下，要在首页添加'立即购买'按钮，点击跳转到商品详情页，对吗？"
    用户: "对的"
    │
    ▼
需求锁定，创建任务
    │
    ▼
pm 在群内同步任务信息
    pm: "任务已创建：t_20260316_001
         标题：首页添加立即购买按钮
         详情：http://localhost:9700/tasks/t_20260316_001
         预计完成时间：30分钟"
    │
    ▼
main 指派给 coder1
    │
    ▼
coder1 开始开发（状态：进行中）
```

## 8. 任务完成流程详解

```
coder1 完成开发
    │
    ├── 1. 提交代码到 Git
    ├── 2. 运行自测
    ├── 3. 生成执行结果文件
    └── 4. 通知 Layer2
    │
    ▼
Layer2 汇总结果
    │
    ├── 1. 收集改动文件列表
    ├── 2. 收集 commit 信息
    ├── 3. 生成任务摘要
    └── 4. 通知 main
    │
    ▼
main 在群内通知
    main: "任务 t_20260316_001 已完成
           改动文件：src/pages/Home/index.tsx
           commit: a1b2c3d
           请在任务详情页确认"
    │
    ▼
状态变更：进行中 → 测试中
    │
    ▼
用户在任务详情页点击「完成」
    │
    ▼
状态变更：测试中 → 已完成
    │
    ├── 1. 归档任务文件
    ├── 2. 向量化存储
    ├── 3. 自动版本升级
    └── 4. 更新统计数据
```

## 9. API 定义

| 方法 | 路径 | 说明 |
|------|------|------|
| `GET` | `/api/v1/tasks` | 获取任务列表（支持分页、状态/优先级筛选、搜索） |
| `GET` | `/api/v1/tasks/:taskId` | 获取任务详情 |
| `POST` | `/api/v1/tasks` | 手动创建任务（管理员通过后台） |
| `PUT` | `/api/v1/tasks/:taskId/status` | 更新任务状态（完成、取消、重新打开） |
| `GET` | `/api/v1/tasks/:taskId/logs` | 获取任务执行日志（Agent 对话、代码变更） |
| `GET` | `/api/v1/tasks/stats` | 获取任务统计（总数、各状态数量、平均耗时） |

## 10. 实现任务清单

> 前置依赖：消息机制模块验收通过。

### 任务 10.1：后端 - 任务管理核心

| # | 任务 | 产出文件 | 说明 |
|---|------|---------|------|
| 1 | 任务数据模型 | `server/src/models/task.ts` | Task 类型定义 + 数据库 CRUD（基于 tasks 表） |
| 2 | 任务管理路由 | `server/src/routes/task.ts` | 上述 6 个 API 端点 |
| 3 | 任务生命周期服务 | `server/src/services/taskLifecycle.ts` | 状态机：pending → confirmed → in_progress → testing → completed/cancelled |
| 4 | 任务文件持久化服务 | `server/src/services/taskFileStore.ts` | 将任务 JSON 写入 inbox/dispatched/completed/suspended/archived 目录 |
| 5 | 任务统计服务 | `server/src/services/taskStats.ts` | 按状态/时间维度的统计聚合查询 |

### 任务 10.2：后端 - 任务记忆化

| # | 任务 | 产出文件 | 说明 |
|---|------|---------|------|
| 1 | 任务摘要生成服务 | `server/src/services/taskSummaryGenerator.ts` | 任务完成后调用中等模型压缩对话和改动 |
| 2 | 任务向量化服务 | `server/src/services/taskVectorStore.ts` | 将任务摘要 Embedding 后存入 ChromaDB task_memory 集合 |
| 3 | 任务检索服务 | `server/src/services/taskSearch.ts` | 语义搜索相似任务，支持关键词 + 向量混合检索 |

### 任务 10.3：前端 - 任务管理页面

| # | 任务 | 产出文件 | 说明 |
|---|------|---------|------|
| 1 | 任务列表页 | `dashboard/src/pages/TaskList/index.tsx` | 筛选（状态/优先级）、搜索、分页 |
| 2 | 任务卡片组件 | `dashboard/src/pages/TaskList/TaskCard.tsx` | 展示标题、状态、优先级、创建时间、参与 Agent |
| 3 | 任务详情页 | `dashboard/src/pages/TaskDetail/index.tsx` | 完整信息：摘要、改动文件、commit、Agent、Token、操作按钮 |
| 4 | 任务执行日志组件 | `dashboard/src/pages/TaskDetail/TaskLogs.tsx` | 时间线展示任务执行过程（Agent 对话、代码变更节点） |
| 5 | 任务统计面板组件 | `dashboard/src/pages/TaskList/TaskStats.tsx` | 总数、各状态数量、平均耗时统计卡片 |
| 6 | 任务 API 封装 | `dashboard/src/services/task.ts` | 封装上述 6 个 API |
| 7 | 状态标签组件 | `dashboard/src/components/TaskStatusBadge.tsx` | pending/in_progress/testing/completed/cancelled 不同颜色 |

## 11. 验收标准

| # | 验收项 | 验证方式 |
|---|--------|---------|
| 1 | `GET /api/v1/tasks` 返回任务列表，分页和筛选正确 | curl 验证 |
| 2 | 任务状态流转正确：创建 → 进行中 → 测试中 → 已完成 | API 连续调用验证 |
| 3 | 任务完成后自动生成摘要并存入 ChromaDB | ChromaDB 查询验证 |

| 7 | 任务执行日志时间线展示正确 | 浏览器截图 |
