## 1. 推荐技术栈

| 层级 | 组件 | 技术方案 |
|------|------|---------|
| **入口层** | 消息接入 | 微信：wechaty / itchat；飞书：OpenClaw 原生 |
| | 消息队列 | Redis Stream 或 BullMQ（Node.js） |
| | 身份鉴权 | 自研（角色/权重/白名单） |
| **调度层** | 核心框架 | CrewAI Flows（任务拆解 + 业务调度 + Prompt 生成） |
| | 上下文压缩 | LangChain + 向量数据库 |
| | 记忆存储 | ChromaDB / Qdrant（本地部署） |
| | 任务持久化 | PostgreSQL + Redis |
| **执行层** | 执行引擎 | OpenClaw Multi-Agent（多实例） |
| | 代码审查 | AutoGen 对话循环（reviewer ↔ coder） |
| | Skills | OpenClaw SkillHub + 自动生成 |
| | 版本管理 | Git（自动 tag + commit） |
| **后台面板** | 前端 | React + TailwindCSS |
| | API | FastAPI / Express |

---

## 2. 模型分级策略

| 级别 | 用途 | 模型 | 成本 |
|------|------|------|------|
| **轻量** | 消息分类、鉴权、队列管理 | DeepSeek-Chat / Kimi | 低 |
| **中等** | 上下文压缩、任务拆解、Prompt 生成 | GPT-4o-mini / DeepSeek-V3 | 中 |
| **强力** | 核心代码编写、架构决策、复杂推理 | Claude Sonnet / GPT-4o | 高 |

## 3. 技术栈详解

### 3.1 Layer1 (WorkBuddy) 技术栈

#### 消息接入

**微信**：
- `wechaty`：Node.js 微信机器人框架
- `itchat`：Python 微信机器人框架（备选）

**飞书**：
- OpenClaw 原生集成
- 飞书开放平台 API

#### 消息队列

**Redis Stream**：
- 轻量级、高性能
- 支持消息持久化
- 支持消费者组

**BullMQ**：
- 基于 Redis 的 Node.js 队列库
- 支持优先级、延迟、重试
- 提供 Web UI 监控

#### 身份鉴权

自研方案：
```javascript
// 基于角色和权重的鉴权系统
const authSystem = {
  roles: ['admin', 'vice_admin', 'employee'],
  weights: { admin: 10, vice_admin: 7, employee: 3 },
  checkPermission: (user, agent) => { /* ... */ }
}
```

---

### 3.2 Layer2 (MiddlewareClaw) 技术栈

#### 核心框架

**CrewAI Flows**：
- 任务拆解和编排
- 业务流程调度
- Prompt 生成

#### 上下文压缩

**LangChain**：
- 文本分割和压缩
- 上下文窗口管理
- Prompt 模板

#### 记忆存储

**ChromaDB**：
- 本地部署的向量数据库
- 支持 Embedding 存储和检索
- Python/JavaScript SDK

**Qdrant**（备选）：
- 高性能向量数据库
- 支持过滤和混合搜索
- Docker 部署

#### 任务持久化

**PostgreSQL**：
- 结构化数据存储
- 任务元数据、用户信息、版本信息
- 支持复杂查询和统计

**Redis**：
- 缓存热数据
- 消息队列
- 会话存储

---

### 3.3 Layer3 (Agents OpenClaw) 技术栈

#### 执行引擎

**OpenClaw Multi-Agent**：
- 多实例隔离
- 独立工作空间
- 共享 Skills 和记忆

#### 代码审查

**AutoGen**：
- 对话循环（reviewer ↔ coder）
- 自动化代码审查
- 问题修复建议

#### Skills

**OpenClaw SkillHub**：
- 预定义 Skills
- 自动生成 Skills
- Skills 共享机制

#### 版本管理

**Git**：
- 自动 commit
- 自动 tag
- 分支管理

---

### 3.4 后台面板技术栈

#### 前端

**React**：
- 组件化开发
- 虚拟 DOM
- 生态丰富

**TailwindCSS**：
- 实用优先的 CSS 框架
- 快速构建 UI
- 响应式设计

#### API

**FastAPI**（Python）：
- 高性能
- 自动生成 API 文档
- 类型检查

**Express**（Node.js，备选）：
- 轻量级
- 中间件丰富
- 易于集成

## 4. 数据库设计

### 4.1 PostgreSQL 表结构

#### 用户表 (users)

```sql
CREATE TABLE users (
  user_id VARCHAR(50) PRIMARY KEY,
  name VARCHAR(100),
  role VARCHAR(20),
  weight INT,
  wechat_id VARCHAR(100),
  feishu_id VARCHAR(100),
  join_date TIMESTAMP,
  created_at TIMESTAMP DEFAULT NOW()
);
```

#### 任务表 (tasks)

```sql
CREATE TABLE tasks (
  task_id VARCHAR(50) PRIMARY KEY,
  title VARCHAR(200),
  status VARCHAR(20),
  priority INT,
  created_by VARCHAR(50),
  created_at TIMESTAMP,
  completed_at TIMESTAMP,
  summary TEXT,
  files_changed JSONB,
  git_commits JSONB,
  agents_involved JSONB,
  token_used INT,
  tags JSONB
);
```

#### 版本表 (versions)

```sql
CREATE TABLE versions (
  version_id VARCHAR(50) PRIMARY KEY,
  project_id VARCHAR(50),
  git_tag VARCHAR(50),
  git_branch VARCHAR(50),
  created_at TIMESTAMP,
  summary TEXT,
  commits JSONB,
  related_tasks JSONB,
  files_changed_count INT,
  build_status VARCHAR(20),
  build_artifact_url VARCHAR(500),
  token_used INT
);
```

## 7. 实现任务清单

## 8. 验收标准

完成以上任务后，必须满足以下条件才算阶段 1（技术选型部分）通过：

| # | 验收项 | 验证方式 |
|---|--------|---------|
| 1 | `npm install` 在根目录和子目录均成功 | 终端执行 |
| 2 | `npm run dev`（server）启动后，访问 `http://localhost:9700/api/v1/health` 返回 `{ "code": 0, "data": { "status": "ok" } }` | 浏览器/curl |
| 3 | `npm run dev`（dashboard）启动后，浏览器可见侧边栏布局骨架 | 浏览器截图 |
| 4 | `docker-compose up -d` 成功启动 PostgreSQL、Redis、ChromaDB | `docker ps` 验证 |
| 5 | 数据库迁移脚本执行成功，四张表创建完成 | `psql` 查询 |
| 6 | 前端页面布局正确，间距和配色符合 TailwindCSS 设计 | 浏览器样式检查 |
