## 1. 目标

建立角色体系和权限规则，确保不同等级的人员只能访问对应的 Agent 和功能。

---

## 2. 角色体系

| 角色 | 权重 | 可交互 Agent | 权限 |
|------|------|-------------|------|
| **管理员** | 10 | main、pm、所有 | 发布任务、配置后台、所有操作 |
| **副管理员** | 7 | main、pm | 发布任务、查看所有信息 |
| **普通员工** | 3 | pm（仅辅助能力） | 查询任务、查看文档、使用辅助工具 |

---

## 3. 权限规则

- 管理员和副管理员可与 main 沟通、发布任务
- 普通员工 @main 时回复：「正在忙，抱歉」
- 所有角色均可与 pm 沟通（普通员工仅限辅助能力）
- 管理员指令优先级 > 副管理员 > 普通员工
- 人员信息通过后台配置，存储到数据库并形成记忆

---

## 4. 群聊规则

- **禁止私聊 Agent**：所有交互必须在群聊中进行
- **群聊记录保存**：全量保存并向量化
- **提供搜索工具**：可快速搜索群聊内容

---

## 5. 核心机制：人员记忆化

## 6. Agent 遵循规范（固化代码）

设计一套 Agent 遵循的规范，固化为代码执行：

```python
# 权限检查规范（固化代码）
def check_permission(user, agent):
    """检查用户是否有权限与指定 Agent 交互"""
    if agent == "main":
        # 只有管理员和副管理员可以与 main 交互
        return user.role in ["admin", "vice_admin"]
    elif agent == "pm":
        # 所有人都可以与 pm 沟通
        return True
    return False

def get_pm_capability(user):
    """获取用户与 pm 交互时的能力范围"""
    if user.role in ["admin", "vice_admin"]:
        return "full"  # 所有功能
    else:
        return "assistant_only"  # 仅辅助能力
```

## 8. 权限验证流程

```
用户发送消息 @Agent
    │
    ▼
Layer1 接收消息
    │
    ├── 1. 识别用户身份（微信/飞书 ID）
    ├── 2. 查询数据库获取用户角色和权重
    ├── 3. 调用 check_permission(user, agent)
    │
    ├── 有权限 ──→ 继续处理消息
    │
    └── 无权限 ──→ 回复「正在忙，抱歉」或「您没有权限」
```

## 9. API 定义

| 方法 | 路径 | 说明 |
|------|------|------|
| `GET` | `/api/v1/users` | 获取人员列表（支持分页、角色筛选） |
| `GET` | `/api/v1/users/:userId` | 获取人员详情 |
| `POST` | `/api/v1/users` | 添加人员 |
| `PUT` | `/api/v1/users/:userId` | 编辑人员信息 |
| `DELETE` | `/api/v1/users/:userId` | 删除人员 |
| `POST` | `/api/v1/auth/check` | 权限校验（传入 userId + agentName，返回是否有权限） |

**POST /api/v1/users 请求示例**

```json
{
  "name": "张三",
  "role": "admin",
  "weight": 10,
  "wechatId": "wxid_zhangsan",
  "feishuId": null,
  "remark": "项目负责人"
}
```

**POST /api/v1/auth/check 请求/响应示例**

```json
// 请求
{ "userId": "user_001", "agent": "main" }

// 响应
{
  "code": 0,
  "data": {
    "allowed": true,
    "role": "admin",
    "capability": "full"
  },
  "message": "ok"
}
```
