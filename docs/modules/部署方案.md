## 2. 安装流程

### 2.1 系统要求

- **操作系统**：macOS / Linux / Windows
- **Node.js**：>= 18.0.0
- **Python**：>= 3.10
- **Git**：>= 2.30
- **Docker**：>= 20.10（可选，用于数据库）

### 2.2 安装步骤

```bash
# 1. 安装 TeamClaw CLI
npm install -g @teamclaw/cli

# 2. 初始化工作空间
teamclaw init

# 3. 安装依赖
cd ~/.openclaw
npm install

# 4. 启动数据库（Docker 方式）
docker-compose up -d

# 5. 启动后台服务
teamclaw serve
# → 浏览器打开 http://localhost:9700

# 6. 导入项目
# 在 Web 后台选择项目路径，开始初始化
```

## 4. Docker Compose 配置

`~/.openclaw/docker-compose.yml`

```yaml
version: '3.8'

services:
  postgres:
    image: postgres:15
    container_name: teamclaw-postgres
    environment:
      POSTGRES_DB: teamclaw
      POSTGRES_USER: teamclaw
      POSTGRES_PASSWORD: your_password
    ports:
      - "5432:5432"
    volumes:
      - postgres_data:/var/lib/postgresql/data

  redis:
    image: redis:7
    container_name: teamclaw-redis
    ports:
      - "6379:6379"
    volumes:
      - redis_data:/data

  chromadb:
    image: chromadb/chroma:latest
    container_name: teamclaw-chromadb
    ports:
      - "8000:8000"
    volumes:
      - chromadb_data:/chroma/chroma

volumes:
  postgres_data:
  redis_data:
  chromadb_data:
```

## 6. 启动服务

### 6.1 开发模式

```bash
# 启动数据库
docker-compose up -d

# 启动后台服务（开发模式，支持热重载）
teamclaw serve --dev

# 启动前端（开发模式）
cd ~/.openclaw/dashboard
npm run dev
```

### 6.2 生产模式

```bash
# 启动数据库
docker-compose up -d

# 构建前端
cd ~/.openclaw/dashboard
npm run build

# 启动后台服务（生产模式，使用 PM2 守护）
pm2 start teamclaw --name teamclaw-server

# 使用 Nginx 托管前端静态文件
# 配置见下文
```

## 7. Nginx 配置

`/etc/nginx/sites-available/teamclaw`

```nginx
server {
    listen 80;
    server_name teamclaw.local;

    # 前端静态文件
    location / {
        root /home/user/.openclaw/dashboard/dist;
        try_files $uri $uri/ /index.html;
    }

    # API 代理
    location /api {
        proxy_pass http://localhost:9700;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }

    # WebSocket 代理
    location /ws {
        proxy_pass http://localhost:9700;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "Upgrade";
        proxy_set_header Host $host;
    }
}
```

## 9. 数据库初始化

```bash
# 创建数据库表
teamclaw db:migrate

# 或手动执行 SQL
psql -U teamclaw -d teamclaw -f ~/.openclaw/migrations/init.sql
```

`~/.openclaw/migrations/init.sql`

```sql
-- 用户表
CREATE TABLE users (
  user_id VARCHAR(50) PRIMARY KEY,
  name VARCHAR(100),
  role VARCHAR(20),
  weight INT,
  wechat_id VARCHAR(100),
  feishu_id VARCHAR(100),
  join_date TIMESTAMP,
  created_at TIMESTAMP DEFAULT NOW()
);

-- 任务表
CREATE TABLE tasks (
  task_id VARCHAR(50) PRIMARY KEY,
  title VARCHAR(200),
  status VARCHAR(20),
  priority INT,
  created_by VARCHAR(50),
  created_at TIMESTAMP,
  completed_at TIMESTAMP,
  summary TEXT,
  files_changed JSONB,
  git_commits JSONB,
  agents_involved JSONB,
  token_used INT,
  tags JSONB
);

-- 版本表
CREATE TABLE versions (
  version_id VARCHAR(50) PRIMARY KEY,
  project_id VARCHAR(50),
  git_tag VARCHAR(50),
  git_branch VARCHAR(50),
  created_at TIMESTAMP,
  summary TEXT,
  commits JSONB,
  related_tasks JSONB,
  files_changed_count INT,
  build_status VARCHAR(20),
  build_artifact_url VARCHAR(500),
  token_used INT
);

-- 项目表
CREATE TABLE projects (
  project_id VARCHAR(50) PRIMARY KEY,
  name VARCHAR(100),
  path VARCHAR(500),
  tech_stack JSONB,
  framework VARCHAR(100),
  build_tool VARCHAR(100),
  summary TEXT,
  created_at TIMESTAMP,
  last_updated TIMESTAMP
);
```

## 11. 备份与恢复

### 11.1 数据库备份

```bash
# 备份 PostgreSQL
pg_dump -U teamclaw teamclaw > backup_$(date +%Y%m%d).sql

# 备份 Redis
redis-cli --rdb /path/to/backup.rdb

# 备份 ChromaDB
docker exec teamclaw-chromadb tar -czf /backup/chromadb.tar.gz /chroma/chroma
```

### 11.2 数据恢复

```bash
# 恢复 PostgreSQL
psql -U teamclaw teamclaw < backup_20260316.sql

# 恢复 Redis
redis-cli --rdb /path/to/backup.rdb

# 恢复 ChromaDB
docker exec teamclaw-chromadb tar -xzf /backup/chromadb.tar.gz -C /
```

## 12. 故障排查

### 12.1 常见问题

| 问题 | 解决方案 |
|------|---------|
| 无法连接数据库 | 检查 Docker 容器是否运行：`docker ps` |
| 端口被占用 | 修改配置文件中的端口号 |
| Agent 无响应 | 检查日志文件，查看错误信息 |
| Token 超限 | 检查 API Key 配额，或切换模型 |

### 12.2 调试模式

```bash
# 启动调试模式
DEBUG=* teamclaw serve

# 查看详细日志
teamclaw serve --verbose
```

## 13. 实现任务清单

| # | 任务 | 产出文件 | 说明 |

| 1 | 编写 docker-compose.yml | `apps/teamclaw/docker-compose.yml` | 与技术选型 7.4 共享，确保 PostgreSQL/Redis/ChromaDB 可一键启动 |
| 2 | 编写启动脚本 | `apps/teamclaw/scripts/dev.sh` | 一键启动 docker + server + dashboard 的开发模式 |

| 4 | 编写 .env.example | `apps/teamclaw/.env.example` | 所有环境变量占位，含注释说明 |

### 任务 13.2：配置文件与环境管理

| # | 任务 | 产出文件 | 说明 |
|---|------|---------|------|

## 14. 验收标准

| # | 验收项 | 验证方式 |
|---|--------|---------|
| 1 | 执行 `scripts/dev.sh` 可一键启动全部开发服务 | 终端执行 |
| 2 | Docker 三个容器均处于 running 状态 | `docker ps` |
| 3 | 数据库初始化脚本幂等执行无报错 | 多次执行 `scripts/db-init.sh` |
| 4 | `GET /api/v1/health` 返回所有服务状态为 ok | curl 验证 |
| 5 | 前端监控页面正确展示各服务状态，样式正常 | 浏览器截图 |
| 6 | PM2 配置和 Nginx 配置模板语法正确 | `pm2 start --dry-run`、`nginx -t` |
