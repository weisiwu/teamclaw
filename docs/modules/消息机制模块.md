# 消息机制模块

[← 返回主文档](../系统架构.V1.md)

---

## 1. 消息处理规则

| 消息类型 | 处理方式 |
|---------|---------|
| **@Agent 的文本消息** | 触发任务流程 |
| **语音消息** | 暂不支持，忽略 |
| **文件消息** | 存储 + 文件理解（仅支持特定文档类型） |
| **表情消息** | 忽略 |
| **Agent 响应前的消息** | 进入消息队列，立即回复「消息已收到，稍后处理」 |

---

## 2. 消息队列与优先级

### 2.1 优先级计算公式

```
优先级 = 角色权重 × 紧急度

紧急度判定：
  包含"紧急/立刻/马上" → ×3
  普通消息             → ×1
```

### 2.2 示例

| 用户 | 角色 | 权重 | 消息内容 | 紧急度 | 优先级 |
|------|------|------|---------|--------|--------|
| 张三 | 管理员 | 10 | "紧急修复首页Bug" | 3 | 30 |
| 李四 | 副管理员 | 7 | "添加一个按钮" | 1 | 7 |
| 王五 | 普通员工 | 3 | "查看任务列表" | 1 | 3 |

---

## 3. 抢占规则

```
抢占规则：
  新任务优先级 > 当前任务优先级 × 1.5 → 触发抢占
  被抢占任务 → tasks/suspended/ 队列
  抢占时群发通知，告知其他人的任务已被挂起
```

### 3.1 抢占示例

```
当前正在执行：李四的任务（优先级 7）

新消息到达：张三的紧急任务（优先级 30）

判断：30 > 7 × 1.5 (10.5) → 触发抢占

操作：
  1. 暂停李四的任务 → tasks/suspended/
  2. 开始执行张三的任务
  3. 群发通知："李四的任务已被暂停，正在处理张三的紧急任务"
```

## 4. 消息合并

同一用户在 **5 分钟内**的连续消息自动合并为一条结构化指令，避免碎片化触发。

### 4.1 合并示例

```
10:00:00 - 用户: "把首页的按钮"
10:00:15 - 用户: "改成蓝色"
10:00:30 - 用户: "字号调大一点"

合并后（10:05:30）：
  "把首页的按钮改成蓝色，字号调大一点"
```

## 6. 文件消息处理

### 6.1 支持的文档类型

| 文档类型 | 处理方式 |
|---------|---------|
| **PPT/PPTX** | 提取文字和图片，转换为 Markdown |
| **Word/DOCX** | 提取文字和表格，转换为 Markdown |
| **Excel/XLSX** | 提取表格数据，转换为 CSV/JSON |
| **PDF** | OCR 识别文字，转换为文本 |
| **图片** | OCR 识别文字，存储原图 |
| **Markdown** | 直接存储 |

### 6.2 文件存储路径

```
~/.openclaw/files/{project_name}/
  ├── documents/     # 文档文件
  ├── images/        # 图片文件
  └── converted/     # 转换后的文件
```

## 7. 消息队列管理流程

```
消息到达
    │
    ├── 1. 识别用户身份和角色
    ├── 2. 计算优先级
    ├── 3. 检查是否需要合并
    ├── 4. 入队
    ├── 5. 检查是否需要抢占
    │
    ├── 需要抢占 ──→ 暂停当前任务，执行新任务
    │
    └── 不需要抢占 ──→ 按优先级排队等待
```
