## 3. 自动版本升级

任务完成后自动 bump 版本号：

### 3.1 版本号规则

```
主版本号.次版本号.修订号

例如：v1.2.3
  1 - 主版本号（重大架构变更）
  2 - 次版本号（新功能）
  3 - 修订号（Bug 修复、小改动）
```

### 3.2 自动升级规则

| 任务类型 | 版本升级规则 |
|---------|-------------|
| **新功能** | 次版本号 +1，修订号归零 |
| **Bug 修复** | 修订号 +1 |
| **重大重构** | 主版本号 +1，次版本号和修订号归零 |

## 4.5 版本对比

支持两个版本之间的变更对比：

```
GET /api/v1/versions/compare?from=v1.0.0&to=v1.1.0
GET /api/v1/versions/compare/quick?from=v1.0.0&to=v1.1.0
```

对比结果包含：commit 数量差、文件变更数、变更类型分布（feat/fix/docs 等）、TOP 变更文件。

**变更统计：**

```
GET /api/v1/versions/change-stats?tag=v1.2.0
```

返回 commit 数、变更类型分布、文件增删行数统计、TOP 5 变更文件。

## 5. 构建流程

```
任务完成 → 自动版本升级
    │
    ├── 1. 创建 git tag
    ├── 2. 触发构建流程
    │
    ▼
构建流程
    │
    ├── 1. 读取项目构建配置
    ├── 2. 执行构建命令（npm run build / gradle build 等）
    ├── 3. 收集构建产物
    ├── 4. 上传到产物存储目录
    ├── 5. 更新版本状态
    │
    ├── 成功 ──→ build_status: "success"
    │
    └── 失败 ──→ build_status: "failed"，记录错误日志
```

## 6. 版本回退

支持恢复到任意历史版本：

```
用户在版本详情页点击「回退到此版本」
    │
    ├── 1. 确认对话框：「确定要回退到 v1.2.0 吗？当前未提交的改动将丢失」
    ├── 2. 用户确认
    │
    ▼
执行回退
    │
    ├── 1. git checkout {tag}
    ├── 2. 创建新分支（可选）
    ├── 3. 更新工作区
    ├── 4. 群发通知：「项目已回退到 v1.2.0」
    │
    ▼
回退完成
```

```
用户在版本详情页点击「创建分支」
    │
    ├── 输入分支名称：[feature/new-ui]
    ├── 选择基于版本：v1.2.1
    │
    ▼
执行创建
    │
    ├── 1. git checkout -b feature/new-ui v1.2.1
    ├── 2. 记录分支信息到数据库
    ├── 3. 群发通知：「已创建分支 feature/new-ui」
    │
    ▼

```

### 7.2 指定主分支

```
用户在后台管理平台选择主分支
    │
    ├── 当前主分支：main
    ├── 可选分支：[main, develop, feature/new-ui]
    ├── 选择：develop
    │
    ▼
更新主分支
    │
    ├── 1. 更新配置文件
    ├── 2. 后续构建基于新主分支
    │
    ▼
更新完成
```

## 8. 变更追踪

版本关联消息截图和变更摘要：

### 8.1 消息截图

每个版本自动关联相关的群聊消息截图：

```
~/.openclaw/versions/{version_id}/
  ├── screenshots/
  │   ├── msg_001.png
  │   ├── msg_002.png
  │   └── ...
  └── summary.md
```

### 8.2 变更摘要

自动生成版本变更摘要：

```markdown
# v1.2.1 变更摘要

## 新功能
- 添加收藏功能，用户可以收藏喜欢的诗词

## Bug 修复
- 修复首页加载性能问题
- 修复登录页面样式错乱

## 改进
- 优化首页按钮样式
- 提升整体响应速度

## 技术改动
- 升级 React Native 到 0.73
- 引入 Zustand 状态管理

## 关联任务
- t_20260316_001: 首页按钮样式修改
- t_20260316_002: 添加收藏功能
```

### 8.3 时间线（Timeline）

版本变更时间线是统一的事件记录入口，基于 `version_change_events` 表：

```
版本创建
    │
    ├── 截图关联（screenshot_linked）
    ├── 变更日志生成（changelog_generated）
    ├── 版本回退（version_rollback）
    └── 手动备注（manual_note）← 可新增/编辑/删除
```

**SSE 实时流**：支持通过 `GET /api/v1/versions/:id/timeline/stream` 建立 SSE 连接，实时接收版本事件推送。

**手动备注**：用户可通过 `POST /api/v1/versions/:id/events` 添加文字备注到时间线，支持后续编辑和删除。

## 9. 版本摘要向量化

版本摘要也需要向量化存储，支持后续检索：

```
版本发布
    │
    ├── 1. 生成版本摘要
    ├── 2. Embedding 向量化
    ├── 3. 存入向量数据库（ChromaDB）
    │
    ▼
支持检索
    │
    ├── "哪个版本加了收藏功能？"
    ├── "上次稳定版本是哪个？"
    └── "这个 commit 属于哪个版本？"

自然语言检索
    │
    ├── POST /api/v1/versions/:id/vector   # 存入向量库
    ├── GET  /api/v1/versions/vector/search?q="哪些版本做了性能优化"
    └── DELETE /api/v1/versions/:id/vector # 删除向量记录
```

## 10. API 定义

### 10.1 核心版本 API

| 方法 | 路径 | 说明 |
|------|------|------|
| `GET` | `/api/v1/versions` | 获取版本列表（支持分页、状态/分支/截图/摘要筛选） |
| `GET` | `/api/v1/versions/search` | 高级搜索（关键词、状态、构建状态、标签、日期范围） |
| `GET` | `/api/v1/versions/:id` | 获取版本详情（摘要、commit、关联任务、构建状态） |
| `POST` | `/api/v1/versions` | 创建新版本 |
| `PUT` | `/api/v1/versions/:id` | 更新版本（含自动 bump 逻辑，仅管理员） |
| `DELETE` | `/api/v1/versions/:id` | 删除版本（需要项目权限） |
| `POST` | `/api/v1/versions/:id/build` | 触发构建（成功自动 bump，自动生成摘要） |
| `GET` | `/api/v1/versions/:id/artifacts` | 获取构建产物列表 |
| `GET` | `/api/v1/versions/:id/artifacts/*` | 下载指定构建产物 |
| `DELETE` | `/api/v1/versions/:id/artifacts` | 删除所有构建产物（仅管理员） |

### 10.2 版本生命周期 API

| 方法 | 路径 | 说明 |
|------|------|------|
| `POST` | `/api/v1/versions/:id/bump` | 手动 bump（自动生成摘要） |
| `POST` | `/api/v1/versions/:id/bump-with-task` | 根据任务类型 bump（patch/minor/major） |
| `POST` | `/api/v1/versions/:id/auto-bump` | 创建新的 bumped 版本记录 |
| `GET` | `/api/v1/versions/:id/bump-preview` | 预览各 bump 类型的结果 |
| `GET` | `/api/v1/versions/:id/bump-history` | 获取 bump 历史 |
| `POST` | `/api/v1/versions/:id/publish` | 发布版本（触发 auto-bump + 自动生成摘要） |
| `GET` | `/api/v1/versions/settings` | 获取版本设置 |
| `PUT` | `/api/v1/versions/settings` | 更新版本设置（autoBump、bumpType、autoTag 等） |

### 10.3 回退 API

| 方法 | 路径 | 说明 |
|------|------|------|
| `POST` | `/api/v1/versions/:id/rollback` | 回退到指定 tag/branch/commit（需项目权限） |
| `GET` | `/api/v1/versions/:id/rollback-targets` | 获取可用的回退目标列表 |
| `GET` | `/api/v1/versions/:id/rollback-preview` | 预览回退效果（ref 查询参数） |
| `GET` | `/api/v1/versions/:id/rollback-history` | 获取回退历史记录 |
| `GET` | `/api/v1/versions/:id/head-status` | 检查版本是否在当前 HEAD |
| `GET` | `/api/v1/versions/compare` | 对比两个版本的详细差异 |
| `GET` | `/api/v1/versions/compare/quick` | 快速对比摘要（from/to 查询参数） |

| 6 | 版本 API 封装 | `lib/api/versions.ts` | 封装所有版本相关 API |
