# 版本管理模块

[← 返回主文档](../系统架构.V1.md)

---

> ⚠️ **数据源要求**：严格禁止使用 mock 数据或伪造数据。所有数据必须来自真实项目仓库（默认 https://github.com/weisiwu/teamclaw ）。数据源可在 [项目导入模块](./项目导入模块.md) 中通过「URL 下载」或「本地上传」方式切换。
>
> 🎨 **样式检测要求**：每个功能在实现过程中，必须检测并验证实际渲染的 UI 样式（布局、间距、配色、响应式等），确保页面展示效果与设计一致，禁止只关注逻辑而忽略样式。

## 1. 功能清单

| 功能 | 说明 |
|------|------|
| **自动 Tag** | 每个特定版本自动创建 git tag |
| **版本面板** | 展示所有 tag 的信息和提交内容 |
| **构建** | 支持一键打包、重新打包 |
| **产物下载** | 提供构建产物的下载入口 |
| **版本回退** | 支持恢复到任意历史版本 |
| **分支管理** | 创建分支、指定主分支 |
| **自动升级** | 任务完成后自动 bump 版本号 |
| **变更追踪** | 版本关联消息截图、变更摘要 |
| **版本摘要** | 自动生成版本摘要并存入数据库 |

## 3. 自动版本升级

任务完成后自动 bump 版本号：

### 3.1 版本号规则

```
主版本号.次版本号.修订号

例如：v1.2.3
  1 - 主版本号（重大架构变更）
  2 - 次版本号（新功能）
  3 - 修订号（Bug 修复、小改动）
```

### 3.2 自动升级规则

| 任务类型 | 版本升级规则 |
|---------|-------------|
| **新功能** | 次版本号 +1，修订号归零 |
| **Bug 修复** | 修订号 +1 |
| **重大重构** | 主版本号 +1，次版本号和修订号归零 |

## 5. 构建流程

```
任务完成 → 自动版本升级
    │
    ├── 1. 创建 git tag
    ├── 2. 触发构建流程
    │
    ▼
构建流程
    │
    ├── 1. 读取项目构建配置
    ├── 2. 执行构建命令（npm run build / gradle build 等）
    ├── 3. 收集构建产物
    ├── 4. 上传到产物存储目录
    ├── 5. 更新版本状态
    │
    ├── 成功 ──→ build_status: "success"
    │
    └── 失败 ──→ build_status: "failed"，记录错误日志
```

## 6. 版本回退

支持恢复到任意历史版本：

```
用户在版本详情页点击「回退到此版本」
    │
    ├── 1. 确认对话框：「确定要回退到 v1.2.0 吗？当前未提交的改动将丢失」
    ├── 2. 用户确认
    │
    ▼
执行回退
    │
    ├── 1. git checkout {tag}
    ├── 2. 创建新分支（可选）
    ├── 3. 更新工作区
    ├── 4. 群发通知：「项目已回退到 v1.2.0」
    │
    ▼
回退完成
```

## 7. 分支管理

### 7.1 创建分支

```
用户在版本详情页点击「创建分支」
    │
    ├── 输入分支名称：[feature/new-ui]
    ├── 选择基于版本：v1.2.1
    │
    ▼
执行创建
    │
    ├── 1. git checkout -b feature/new-ui v1.2.1
    ├── 2. 记录分支信息到数据库
    ├── 3. 群发通知：「已创建分支 feature/new-ui」
    │
    ▼
创建完成
```

### 7.2 指定主分支

```
用户在后台管理平台选择主分支
    │
    ├── 当前主分支：main
    ├── 可选分支：[main, develop, feature/new-ui]
    ├── 选择：develop
    │
    ▼
更新主分支
    │
    ├── 1. 更新配置文件
    ├── 2. 后续构建基于新主分支
    │
    ▼
更新完成
```

## 8. 变更追踪

版本关联消息截图和变更摘要：

### 8.1 消息截图

每个版本自动关联相关的群聊消息截图：

```
~/.openclaw/versions/{version_id}/
  ├── screenshots/
  │   ├── msg_001.png
  │   ├── msg_002.png
  │   └── ...
  └── summary.md
```

### 8.2 变更摘要

自动生成版本变更摘要：

```markdown
# v1.2.1 变更摘要

## 新功能
- 添加收藏功能，用户可以收藏喜欢的诗词

## Bug 修复
- 修复首页加载性能问题
- 修复登录页面样式错乱

## 改进
- 优化首页按钮样式
- 提升整体响应速度

## 技术改动
- 升级 React Native 到 0.73
- 引入 Zustand 状态管理

## 关联任务
- t_20260316_001: 首页按钮样式修改
- t_20260316_002: 添加收藏功能
```

## 9. 版本摘要向量化

版本摘要也需要向量化存储，支持后续检索：

```
版本发布
    │
    ├── 1. 生成版本摘要
    ├── 2. Embedding 向量化
    ├── 3. 存入向量数据库
    │
    ▼
支持检索
    │
    ├── "哪个版本加了收藏功能？"
    ├── "上次稳定版本是哪个？"
    └── "这个 commit 属于哪个版本？"
```
