## 3. 自动版本升级

任务完成后自动 bump 版本号：

### 3.1 版本号规则

```
主版本号.次版本号.修订号

例如：v1.2.3
  1 - 主版本号（重大架构变更）
  2 - 次版本号（新功能）
  3 - 修订号（Bug 修复、小改动）
```

### 3.2 自动升级规则

| 任务类型 | 版本升级规则 |
|---------|-------------|
| **新功能** | 次版本号 +1，修订号归零 |
| **Bug 修复** | 修订号 +1 |
| **重大重构** | 主版本号 +1，次版本号和修订号归零 |

## 5. 构建流程

```
任务完成 → 自动版本升级
    │
    ├── 1. 创建 git tag
    ├── 2. 触发构建流程
    │
    ▼
构建流程
    │
    ├── 1. 读取项目构建配置
    ├── 2. 执行构建命令（npm run build / gradle build 等）
    ├── 3. 收集构建产物
    ├── 4. 上传到产物存储目录
    ├── 5. 更新版本状态
    │
    ├── 成功 ──→ build_status: "success"
    │
    └── 失败 ──→ build_status: "failed"，记录错误日志
```

## 6. 版本回退

支持恢复到任意历史版本：

```
用户在版本详情页点击「回退到此版本」
    │
    ├── 1. 确认对话框：「确定要回退到 v1.2.0 吗？当前未提交的改动将丢失」
    ├── 2. 用户确认
    │
    ▼
执行回退
    │
    ├── 1. git checkout {tag}
    ├── 2. 创建新分支（可选）
    ├── 3. 更新工作区
    ├── 4. 群发通知：「项目已回退到 v1.2.0」
    │
    ▼
回退完成
```

## 7. 分支管理

### 7.1 创建分支

```
用户在版本详情页点击「创建分支」
    │
    ├── 输入分支名称：[feature/new-ui]
    ├── 选择基于版本：v1.2.1
    │
    ▼
执行创建
    │
    ├── 1. git checkout -b feature/new-ui v1.2.1
    ├── 2. 记录分支信息到数据库
    ├── 3. 群发通知：「已创建分支 feature/new-ui」
    │
    ▼
创建完成
```

### 7.2 指定主分支

```
用户在后台管理平台选择主分支
    │
    ├── 当前主分支：main
    ├── 可选分支：[main, develop, feature/new-ui]
    ├── 选择：develop
    │
    ▼
更新主分支
    │
    ├── 1. 更新配置文件
    ├── 2. 后续构建基于新主分支
    │
    ▼
更新完成
```

## 8. 变更追踪

版本关联消息截图和变更摘要：

### 8.1 消息截图

每个版本自动关联相关的群聊消息截图：

```
~/.openclaw/versions/{version_id}/
  ├── screenshots/
  │   ├── msg_001.png
  │   ├── msg_002.png
  │   └── ...
  └── summary.md
```

### 8.2 变更摘要

自动生成版本变更摘要：

```markdown
# v1.2.1 变更摘要

## 新功能
- 添加收藏功能，用户可以收藏喜欢的诗词

- t_20260316_001: 首页按钮样式修改
```

## 9. 版本摘要向量化

版本摘要也需要向量化存储，支持后续检索：

```
版本发布
    │
    ├── 1. 生成版本摘要
    ├── 2. Embedding 向量化
    ├── 3. 存入向量数据库
    │
    ▼
支持检索
    │
    ├── "哪个版本加了收藏功能？"
    ├── "上次稳定版本是哪个？"
    └── "这个 commit 属于哪个版本？"
```

## 10. API 定义

| 方法 | 路径 | 说明 |
|------|------|------|
| `GET` | `/api/v1/versions` | 获取版本列表（支持分页、按分支筛选） |
| `GET` | `/api/v1/versions/:versionId` | 获取版本详情（摘要、commit、关联任务、构建状态） |
| `POST` | `/api/v1/versions/:versionId/build` | 触发构建 |
| `POST` | `/api/v1/versions/:versionId/rebuild` | 重新构建 |
| `GET` | `/api/v1/versions/:versionId/artifact` | 下载构建产物 |
| `POST` | `/api/v1/versions/:versionId/rollback` | 回退到指定版本 |
| `GET` | `/api/v1/branches` | 获取分支列表 |
| `POST` | `/api/v1/branches` | 创建分支（基于指定版本） |
| `PUT` | `/api/v1/branches/primary` | 指定主分支 |

**GET /api/v1/versions 响应示例**

```json
{
  "code": 0,
  "data": {
    "list": [
      {
        "versionId": "v1.2.1",
        "gitTag": "v1.2.1",
        "gitBranch": "main",
        "createdAt": "2026-03-16T15:00:00+08:00",
        "summary": "新增收藏功能、修复首页加载性能问题",
        "commitCount": 3,
        "relatedTaskCount": 2,
        "buildStatus": "success",
        "tokenUsed": 15000
      }
    ],
    "total": 10
  },
  "message": "ok"
}
```

**POST /api/v1/versions/:versionId/rollback 请求示例**

```json
{
  "createBranch": true,
  "branchName": "rollback/v1.2.0"
}
```

## 12. 验收标准

| # | 验收项 | 验证方式 |
|---|--------|---------|
| 1 | 任务完成后自动创建 git tag，版本号 bump 规则正确 | git log + API 验证 |
| 2 | `GET /api/v1/versions` 返回版本列表，分支筛选正确 | curl 验证 |
| 3 | 版本详情页完整展示 commit、关联任务、改动文件 | 浏览器截图 |
| 4 | 触发构建后产物正确生成，下载链接可用 | 浏览器下载验证 |
| 5 | 重新构建功能正常，覆盖旧产物 | 操作验证 |
| 6 | 版本回退后工作区代码正确切换 | git status 验证 |
| 7 | 分支创建和主分支切换功能正常 | API + git 验证 |
| 8 | 版本摘要已向量化存入 ChromaDB | ChromaDB 查询验证 |
| 9 | 所有页面样式正确，构建状态标签颜色区分，空状态展示正确 | 浏览器样式检查 |
