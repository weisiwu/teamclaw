# 44【P1】设置页移入侧边栏固定位置

## 背景
设置入口当前藏在侧边栏底部折叠按钮旁边，不直观。用户反馈"设置为什么不做成跟侧边栏一样，而要单独收起来"。

## 目标
- 将设置（⚙️）作为侧边栏的固定底部入口，与其他菜单项同级
- 设置页面整合 Admin 后台功能（配置、审计日志、Webhook、Demo 数据）
- 设置入口在侧边栏折叠/展开状态下都可见

## 技术方案
1. 修改 `Sidebar.tsx`，将设置从底部折叠区域移到 menuItems 列表末尾（分隔线下方）
2. 设置页面新增 Tab：
   - 主题/外观（原有）
   - 系统配置（原 `/admin/config`）
   - 审计日志（原 `/admin/audit`）
   - Webhook（原 `/admin/webhooks`）
   - 定时任务（原 `/cron`，从侧边栏移除独立入口）
3. 原 `/admin/*` 路由做 redirect 到 `/settings`
4. 原 `/cron` 路由做 redirect 到 `/settings`

## 实现文件
- `components/layout/Sidebar.tsx` — 调整设置入口位置
- `app/settings/page.tsx` — 整合 Admin 功能
- `app/admin/*/page.tsx` — 改为 redirect
- `app/cron/page.tsx` — 改为 redirect

## 依赖关系
- 前置：—
- 后续：45【P1】非核心功能页面精简与合并

## 验证方式
1. 设置在侧边栏底部常驻，折叠态也可见
2. 设置页 Tab 切换正常，原 Admin 功能无丢失
3. 原 `/admin/*` 和 `/cron` 路由正确跳转
4. 权限控制仍然有效（Admin Tab 仅管理员可见）
5. **至少 5 轮修改验证**：入口位置→Tab切换→权限→redirect→暗色模式
