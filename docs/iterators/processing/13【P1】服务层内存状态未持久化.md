# 04【P1】服务层内存状态未持久化

> 优先级：🟠 P1（数据丢失风险）
> 发现日期：2026-03-25
> 状态：待处理

---

## 问题描述

虽然部分服务（如 `userService`、`cronService`）已迁移到 PostgreSQL，但仍有大量服务使用 **内存 Map/数组** 存储运行时状态，服务重启后数据全部丢失。

## 风险

- 服务重启后 **所有运行时数据丢失**：任务状态、消息队列、Agent 执行上下文、Token 统计等
- 多进程/集群部署时数据不共享
- Token 消费统计不可靠（重启后清零）

## 优化方案

### 分批迁移（按重要性）

**第一批（核心业务数据）**：
- `taskLifecycle.ts` → PostgreSQL `tasks` 表
- `messageQueue.ts` → PostgreSQL + Redis 缓存
- `importOrchestrator.ts` → PostgreSQL `import_tasks` 表
- `tokenStatsService.ts` → PostgreSQL `token_usage` 表

**第二批（运行时状态）**：
- `agentExecution.ts` → PostgreSQL `agent_executions` 表
- `taskMemory.ts` → PostgreSQL + 向量化
- `taskNotification.ts` → PostgreSQL `webhook_endpoints`, `notification_rules` 表

**第三批（辅助功能）**：
- `messageCircuitBreaker.ts` → Redis（适合短期状态）
- `messageRateLimiter.ts` → Redis
- `messageDLQ.ts` → PostgreSQL（需要持久化）
- 其余统计/缓存类 → Redis

## 验收标准

- [ ] 核心业务数据（任务、消息、导入、Token 统计）持久化到 PostgreSQL
- [ ] 服务重启后数据不丢失
- [ ] 短期状态（限流、熔断）使用 Redis
- [ ] 内存 Map 仅用于进程内缓存，有 DB 作为持久层
