# 35【P0】删除误创建的 --version 目录

## 问题描述

项目根目录下存在一个名为 `--version` 的异常目录：

```
apps/teamclaw/--version/
  └── _/    (空目录)
```

该目录为命令行误操作产生（推测执行了类似 `mkdir --version` 的命令），内部仅包含一个空的 `_/` 子目录，无任何有效内容。

## 影响

- 目录名以 `--` 开头，不符合任何命名规范
- 可能导致部分工具或脚本解析路径时出错（`--` 通常被解释为命令行参数）
- 污染项目目录结构

## 修复方案

直接删除该目录：

```bash
rm -rf './--version'
```

注意：由于目录名以 `--` 开头，需要使用 `./` 前缀或 `--` 参数终止符避免被 shell 误解析为参数：

```bash
rm -rf -- --version
```

## 预防措施

建议在 `.gitignore` 中排除此类异常目录名（如果尚未被 Git 跟踪则无需额外操作）。
