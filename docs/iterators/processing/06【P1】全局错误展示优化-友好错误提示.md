# 41【P1】全局错误展示优化 - 友好错误提示

## 背景
大量页面在 API 请求失败时直接将原始错误信息抛给用户，如 `Unexpected token '<', "<!DOCTYPE "... is not valid JSON`。这不是理想的错误展示方式，用户无法理解且体验极差。

## 目标
- 建立统一的全局错误处理机制
- API 请求失败时展示友好的中文提示，而非原始 JSON/HTML 错误
- 区分不同错误类型（网络错误、服务端错误、认证失效）给予不同提示
- 提供「重试」按钮

## 技术方案
1. 创建全局 API 错误拦截器（fetch wrapper 或 react-query 的 onError）
2. 错误分类：
   - HTML 响应（`<!DOCTYPE`）→ "服务暂时不可用，请稍后重试"
   - 网络错误（`fetch failed`）→ "网络连接失败，请检查网络"
   - 401/403 → "登录已过期，请重新登录"
   - 500 → "服务器内部错误，请稍后重试"
3. 使用统一的 Toast/Alert 组件展示错误
4. 在错误展示中隐藏技术细节，仅在开发模式下 console.error 输出原始信息

## 实现文件
- `lib/api.ts` 或 `lib/fetch.ts` — 创建统一请求封装
- `components/providers/QueryProvider.tsx` — 配置 react-query 全局 onError
- `components/ui/ErrorAlert.tsx` — 统一错误展示组件
- 各 `page.tsx` — 替换裸 fetch 为统一封装

## 验证方式
1. 模拟服务端关闭，页面展示友好提示而非原始错误
2. 模拟返回 HTML（非 JSON），展示 "服务暂时不可用"
3. 模拟 401，自动跳转登录
4. 所有页面的错误展示风格一致
5. **至少 5 轮修改验证**：网络错误→HTML响应→401→500→全页面巡检
