# 【P0】本地数据库连接配置与开发环境搭建指南

## 背景
本地运行项目时数据库连不上，服务端硬编码 `postgresql://localhost:5432/teamclaw` 无任何提示和降级方案。没有 `.env.example` 文件，开发者无法知道需要配置哪些环境变量。

## 目标
- 创建 `server/.env.example` 文件，列出所有必需环境变量
- 数据库连接失败时给出友好错误提示（而非直接崩溃）
- 编写本地开发环境一键搭建脚本（自动创建数据库、运行迁移）
- 补充 README 中的本地开发指南

## 技术方案
1. 创建 `server/.env.example`，包含 DATABASE_URL、PORT、JWT_SECRET 等
2. 修改 `server/src/utils/db.ts`，添加连接失败友好提示和重试逻辑
3. 创建 `scripts/setup-db.sh`，自动执行 createdb + migration
4. 更新项目根 README.md 增加「快速开始」段落

## 实现文件
- `server/.env.example` — 新建
- `server/src/utils/db.ts` — 增加连接错误处理
- `scripts/setup-db.sh` — 新建一键初始化脚本
- `README.md` — 补充本地开发指南

## 验证方式
1. 全新 clone 后按 README 操作可成功启动
2. 未配置数据库时，服务端给出友好错误提示而非堆栈崩溃
3. `setup-db.sh` 可自动创建数据库并运行迁移
4. 环境变量缺失时有明确的日志输出告知缺什么
5. **至少 5 轮修改验证**：.env.example→连接错误处理→setup脚本→README→端到端验证
