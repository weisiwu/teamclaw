# 42【P1】页面宽度与样式全局统一

## 背景
每个 Tab 对应的页面宽度不一致：有的占满剩余所有宽度，有的在两边留有很大边距。这类样式问题需要全局检查一遍并统一修复。

## 目标
- 所有页面使用统一的容器宽度和内边距
- 定义统一的页面容器 CSS 类（如 `page-container`）
- 页面标题、间距、字体大小统一
- 移动端和桌面端的响应式断点一致

## 技术方案
1. 审计所有 page.tsx 的容器样式，列出不一致项
2. 定义统一的页面容器规范（最大宽度、内边距、标题样式）
3. 将所有页面统一使用 `page-container` 类
4. 检查并修复以下样式问题：
   - 页面最大宽度不一致
   - 内边距不统一
   - 标题层级和字号不一致
   - 卡片/表格间距不一致
   - 响应式断点不一致

## 实现文件
- `app/globals.css` 或 `tailwind.config.ts` — 定义统一样式变量
- 所有 `app/*/page.tsx` — 统一容器类名
- `components/layout/AppLayout.tsx` — 可能需要调整主内容区域样式

## 依赖关系

## 验证方式
1. 逐页截图对比，确认所有页面宽度一致
2. 缩小浏览器窗口，验证响应式断点一致
3. 页面标题字号、间距、对齐方式一致
4. 暗色模式下样式不破坏
5. **至少 5 轮修改验证**：桌面宽度→移动端宽度→标题样式→间距→暗色模式
