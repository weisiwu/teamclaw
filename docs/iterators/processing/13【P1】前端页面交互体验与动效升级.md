# 【P1】前端页面交互体验与动效升级

## 背景
当前页面交互生硬，缺乏过渡动画和反馈。页面切换无动效、列表无进入动画、操作无反馈感。整体体验停留在"能用"层面，距离"好用"有差距。

## 目标
- 页面切换增加平滑过渡动画
- 列表项增加错落进入动画（staggered animation）
- 按钮/卡片点击增加反馈效果（涟漪、缩放）
- 加载状态使用骨架屏 + 呼吸灯效果
- 空状态增加插图和引导动画
- 侧边栏切换增加平滑过渡

## 技术方案
1. 引入 Framer Motion（或使用 CSS 动画）实现页面过渡
2. 列表使用 stagger 动画（每项延迟 50ms 进入）
3. 按钮组件增加点击缩放效果（scale 0.95 → 1）
4. 统一骨架屏组件，增加 shimmer 动效
5. 创建空状态插图组件（使用 Lucide 图标组合或 SVG）
6. 侧边栏折叠/展开使用 spring 动画

## 实现文件
- `package.json` — 可选增加 framer-motion
- `app/globals.css` — 新增动效关键帧和工具类
- `components/ui/Skeleton.tsx` — 统一骨架屏组件
- `components/ui/EmptyState.tsx` — 空状态组件
- `components/ui/AnimatedList.tsx` — 动画列表容器
- `components/layout/Sidebar.tsx` — 折叠动画升级
- 各 `page.tsx` — 应用页面过渡

## 验证方式
1. 页面切换有平滑过渡（无生硬跳转）
2. 列表加载时项目依次进入（stagger 效果）
3. 按钮点击有视觉反馈
4. 骨架屏风格统一且有动效
5. **至少 5 轮修改验证**：页面过渡→列表动画→按钮反馈→骨架屏→空状态
