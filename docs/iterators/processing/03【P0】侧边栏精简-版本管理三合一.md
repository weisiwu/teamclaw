# 38【P0】侧边栏精简 - 版本管理三合一

## 背景
当前侧边栏中「版本管理」「版本面板」「分支管理」三个入口本质是同一功能域，拆成 3 个入口增加用户认知负担。

## 目标
- 将 `/versions`、`/versions/panel`、`/branches` 三个页面合并为一个 `/versions` 页面
- 侧边栏只保留一个「版本管理」入口
- 原版本面板内容整合到版本管理主视图
- 原分支管理作为版本管理页的 Tab 或区域

## 技术方案
1. 重构 `/versions/page.tsx`，使用 Tab 组件整合三个页面的内容
2. 从 `Sidebar.tsx` 移除「版本面板」和「分支管理」两个 menuItem
3. 保留原 `/versions/panel` 和 `/branches` 路由做 redirect（兼容）
4. 迁移 `components/versions/` 和 `components/branch/` 中的组件到统一页面

## 实现文件
- `components/layout/Sidebar.tsx` — 移除多余入口
- `app/versions/page.tsx` — 重构为 Tab 页面
- `app/versions/panel/page.tsx` — 改为 redirect
- `app/branches/page.tsx` — 改为 redirect
- `components/versions/*` — 组件整合
- `components/branch/*` — 组件迁移

- 后续：45【P1】非核心功能页面精简与合并

## 验证方式
1. 侧边栏只有一个「版本管理」入口
2. 点击进入后可通过 Tab 切换查看版本列表、版本面板、分支管理
3. 原 `/versions/panel` 和 `/branches` 路由自动跳转到新页面
4. 所有原有功能无丢失
5. **至少 5 轮修改验证**：每轮检查一个维度（布局→数据加载→交互→响应式→暗色模式）
