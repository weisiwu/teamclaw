# 17【P2】加载状态骨架屏风格不统一

## 问题描述

各页面的加载状态展示方式不一致，存在多种实现：

| 页面 | 加载方式 | 问题 |
|------|---------|------|
| `app/page.tsx` | `animate-pulse` 卡片骨架 | ✅ 较规范 |
| `app/versions/page.tsx` | `<Loader2>` 居中旋转 | 无骨架，仅 spinner |
| `app/members/page.tsx` | `<MembersSkeleton>` 组件 | ✅ 已组件化 |
| `app/agent-team/page.tsx` | `<AgentTeamSkeleton>` 组件 | ✅ 已组件化 |
| `app/capabilities/page.tsx` | `<Loader2>` 全屏旋转 | 无骨架，仅 spinner |
| `app/cron/page.tsx` | 文字 "加载中..." | 最简，无视觉反馈 |
| `app/monitor/page.tsx` | `<Loader2>` + Card 居中 | 独特布局，不统一 |
| `app/tokens/page.tsx` | `<TokensLoading>` 骨架组件 | ✅ 已组件化 |
| `app/docs/page.tsx` | 无加载状态 | 缺失 |

## 期望效果

- 列表/表格页面统一使用骨架屏（`animate-pulse` 占位块）
- 简单数据页面统一使用 `page-loading` 工具类（已在 `globals.css` 中定义）
- 所有加载状态应有一致的视觉语言

## 样式方案

**样式类型：Tailwind CSS 工具类 + 共享骨架组件**

### 已有的 `page-loading` 工具类

```css
/* globals.css 中已定义 */
.page-loading {
  @apply flex items-center justify-center py-12 gap-2 text-muted-foreground;
}
```

### 统一加载组件模式

#### 简单页面加载（无骨架）

```tsx
<div className="page-loading">
  <Loader2 className="w-6 h-6 animate-spin" />
  <span>加载中...</span>
</div>
```

#### 列表页骨架屏

复用已有 `components/ui/projects-skeleton.tsx` 中的模式，为缺失骨架屏的页面创建对应骨架：

```tsx
// 通用列表骨架
function ListSkeleton({ rows = 5 }: { rows?: number }) {
  return (
    <div className="space-y-3">
      {[...Array(rows)].map((_, i) => (
        <div key={i} className="animate-pulse">
          <div className="bg-card border border-border rounded-xl p-4">
            <div className="h-4 bg-muted rounded w-1/3 mb-2" />
            <div className="h-3 bg-muted rounded w-2/3" />
          </div>
        </div>
      ))}
    </div>
  );
}
```

## 修改步骤

3. `app/cron/page.tsx`：替换文字为 `page-loading` 样式的 spinner
4. `app/monitor/page.tsx`：使用 `page-loading` 统一布局
5. `app/docs/page.tsx`：添加文档列表加载状态
