# 14【P2】硬编码颜色未使用语义化设计令牌

## 问题描述

项目已在 `tailwind.config.ts` 和 `globals.css` 中建立了完整的语义化设计令牌体系（CSS 变量驱动），但大量页面仍使用 Tailwind 的硬编码颜色类（如 `text-gray-900`、`bg-white`、`border-gray-200`），导致：

1. 深色模式需要手动添加 `dark:` 变体，维护成本高
2. 未来更换主题色时需要逐文件修改
3. 颜色语义不明确，同一灰色在不同地方含义不同

## 已有设计令牌

定义在 `globals.css` `:root` 和 `.dark` 选择器中，通过 `tailwind.config.ts` 映射为 Tailwind 类名：

| 语义令牌 | 浅色值 | 深色值 | 用途 |
|---------|--------|--------|------|
| `text-foreground` | slate-900 | slate-50 | 主文本 |
| `bg-background` | gray-50 | slate-900 | 页面背景 |
| `bg-card` / `text-card-foreground` | white / slate-900 | slate-800 / slate-50 | 卡片 |
| `border-border` | slate-200 | slate-700 | 边框 |
| `text-muted-foreground` | slate-500 | slate-400 | 次要文本 |
| `bg-muted` | slate-100 | slate-800 | 次要背景 |
| `bg-primary` / `text-primary-foreground` | blue-600 | blue-500 | 主操作色 |
| `bg-secondary` / `text-secondary-foreground` | slate-100 | slate-800 | 辅助区块 |
| `bg-destructive` | red-500 | red-400 | 危险操作 |

## 替换对照表

### 高频替换（全局适用）

| 硬编码写法 | 替换为 | 出现位置 |
|-----------|--------|---------|
| `text-gray-900 dark:text-white` | `text-foreground` | 全局标题、正文 |
| `bg-white dark:bg-slate-800` | `bg-card` | 卡片、弹窗背景 |
| `border-gray-200 dark:border-slate-700` | `border-border` | 分割线、卡片边框 |
| `text-gray-500 dark:text-gray-400` | `text-muted-foreground` | 描述文字、时间戳 |
| `bg-gray-50 dark:bg-slate-900` | `bg-background` | 页面背景 |
| `bg-gray-100 dark:bg-slate-700` | `bg-muted` | hover 背景、表头 |
| `text-gray-600 dark:text-gray-300` | `text-muted-foreground` | 二级文本 |
| `text-gray-700 dark:text-gray-200` | `text-foreground` | 表单 label |

### 注意保留的颜色

以下颜色有特定语义含义，不应替换：
- 状态色：`text-green-600`、`text-red-500`、`text-yellow-600`、`text-blue-600`
- 渐变色：`from-blue-50 to-indigo-100` 等装饰用途
- Badge 颜色：由 Badge 组件 variant 控制

## 样式方案

**样式类型：Tailwind CSS 语义化令牌替换**

### 实施策略

1. **批量查找替换**：按上方对照表进行全局查找替换
2. **优先级**：先替换背景色和文本色（影响最大），再替换边框色
3. **验证**：每个文件替换后切换深色模式验证
