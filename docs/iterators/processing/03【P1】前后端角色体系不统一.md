# 03【P1】前后端角色体系不统一

> 优先级：🟠 P1（权限逻辑混乱）
> 发现日期：2026-03-25
> 状态：待处理

---

## 问题描述

项目中存在 **两套完全不同的角色定义**，前后端不统一：

### 后端角色体系（`server/src/constants/roles.ts`）

```typescript
export type Role = "admin" | "vice_admin" | "member";
```

- 3 个角色：管理员、副管理员、普通员工
- 权重：admin=10, vice_admin=7, member=3
- Agent 访问矩阵基于此角色

### 前端角色体系（`lib/auth/roles.ts`）

```typescript
export type Role = 'owner' | 'admin' | 'developer' | 'viewer';
```

- 4 个角色：所有者、管理员、开发者、访客
- 与后端角色无交集（`owner`、`developer`、`viewer` 后端不存在）
- 前端 `lib/api-shared.ts` 中又使用了后端的角色定义 `"admin" | "vice_admin" | "member"`

### 具体冲突点

| 位置 | 角色定义 | 使用场景 |
|------|---------|---------|
| `server/src/constants/roles.ts` | admin / vice_admin / member | JWT Token、权限检查、Agent 访问矩阵 |
| `lib/auth/roles.ts` | owner / admin / developer / viewer | 前端组件、权限判断、角色标签展示 |
| `lib/api-shared.ts` | admin / vice_admin / member | Next.js API Routes 中间件 |

## 风险

- JWT Token 中包含 `role: "admin"`，前端 `lib/auth/roles.ts` 的 `isAdmin()` 检查 `role === 'admin' || role === 'owner'`，但 `owner` 在后端不存在
- 前端组件显示 `开发者`、`访客` 等角色，但后端没有这些概念
- 权限判断逻辑前后端不一致，可能出现前端允许但后端拒绝的操作

## 优化方案

1. **统一为一套角色定义**：选择后端的 `admin / vice_admin / member` 或扩展为新体系
2. 删除或改造 `lib/auth/roles.ts` 以匹配后端
3. 前端组件中的角色标签映射与后端对齐
4. 确保 JWT payload 中的 role 字段在前后端有一致的解析逻辑

## 涉及文件

- `server/src/constants/roles.ts` → 权威角色定义（以此为准或扩展）
- `lib/auth/roles.ts` → 需重写或删除
- `lib/api-shared.ts` → 角色类型需与统一定义对齐
- 所有使用 `lib/auth/roles.ts` 的前端组件

## 验收标准

- [ ] 全项目只有一套 `Role` 类型定义
- [ ] 前端展示的角色标签与后端一致
- [ ] JWT Token 中的 role 在前后端均可正确解析
- [ ] 权限检查前后端行为一致
