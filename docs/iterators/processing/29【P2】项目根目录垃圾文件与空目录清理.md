# 07【P2】项目根目录垃圾文件与空目录清理

> 优先级：🟡 P2（工程整洁）
> 发现日期：2026-03-25
> 状态：待处理

---

## 问题描述

项目根目录存在异常文件和空目录，影响工程整洁性。

### 异常文件

| 文件/目录 | 大小 | 问题 |
|----------|------|------|
| `{` | 0 bytes | 意外创建的空文件，文件名为单个花括号 |
| `--version/` | 0 items | 空目录，疑似 CLI 参数误创建 |
| `.DS_Store` | 8196 bytes | macOS 系统文件，不应入版本控制 |

### 空目录

| 目录 | 说明 |
|------|------|
| `--version/` | 0 items，无任何用途 |

### .gitignore 检查

需确认以下条目是否已在 `.gitignore` 中：
- `.DS_Store`
- `server/dist/`（构建产物，L7 清单项）

## 优化方案

1. 删除 `{` 文件
2. 删除 `--version/` 空目录
3. 确认 `.gitignore` 包含 `.DS_Store`
4. 运行 `git rm -r --cached` 清除已跟踪的无用文件

## 涉及操作

```bash
# 删除垃圾文件
rm -f "{"
rm -rf -- "--version/"

# 确保 .gitignore 包含
echo ".DS_Store" >> .gitignore  # 如果还没有

# 清除已跟踪的文件
git rm --cached .DS_Store 2>/dev/null || true
git rm -r --cached "server/dist/" 2>/dev/null || true
```

## 验收标准

- [ ] `{` 文件已删除
- [ ] `--version/` 目录已删除
- [ ] `.DS_Store` 在 `.gitignore` 中
- [ ] `git status` 无异常未跟踪文件
