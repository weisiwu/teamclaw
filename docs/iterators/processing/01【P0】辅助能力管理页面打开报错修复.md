# 01【P0】辅助能力管理页面打开报错修复

## 问题描述

`/capabilities` 页面打开即报错，页面白屏或显示异常。

## 根因

页面调用 `fetch('/api/v1/abilities')`，但 **不存在对应的 Next.js API 代理路由**。请求返回 HTML 404 页面，`res.json()` 解析 HTML 时抛出 `SyntaxError`。

### 缺失的代理路由

| 后端路由 | Next.js 代理路由（缺失） |
|----------|--------------------------|
| GET `/api/v1/abilities` | `app/api/v1/abilities/route.ts` |
| GET `/api/v1/abilities/:id` | `app/api/v1/abilities/[id]/route.ts` |
| PUT `/api/v1/abilities/:id/toggle` | `app/api/v1/abilities/[id]/toggle/route.ts` |
| POST `/api/v1/abilities/reset` | `app/api/v1/abilities/reset/route.ts` |

## 修复方案

**已修复** — 创建了以上 4 个 Next.js 代理路由文件，使用 `proxyNextToBackend` 转发到 Express 后端。

### 新建文件

- `app/api/v1/abilities/route.ts` — GET 列表
- `app/api/v1/abilities/[id]/route.ts` — GET 单个
- `app/api/v1/abilities/[id]/toggle/route.ts` — PUT 切换状态
- `app/api/v1/abilities/reset/route.ts` — POST 重置

### 验证

```bash
curl -s http://localhost:3000/api/v1/abilities | python3 -c "import sys,json; d=json.load(sys.stdin); print(d['success'], len(d['data']['list']))"
# True 6
```
