# 45【P1】非核心功能页面精简与合并

## 背景
功能要做减法不做加法。非核心功能应该优化，减小用户心智负担。每个非核心功能最好在一个页面内包住。

## 目标
- Tools & Skills（`/capabilities`）合并到 Agent 团队页（`/agent-team`）作为 Tab
- 文档中心（`/docs`）从侧边栏移除（保留路由，但不在侧边栏展示）
- 消息（`/messages`）合并到任务详情中（不再独立存在）
- 监控（`/monitor`）删除或合并到设置
- 标签（`/tags`）降级为任务管理的过滤维度，不再独立页面
- 侧边栏最终只保留 6 个核心入口

## 技术方案
1. Agent 团队页新增 Tab：Tools、Skills（迁移自 `/capabilities`）
2. 从 Sidebar.tsx 移除：定时任务、文档中心、Tools & Skills（这些在前置任务中已部分完成）
3. 标签功能整合到任务管理页的筛选器中
4. 消息功能整合到任务详情页的消息 Tab 中
5. 清理不再需要的独立路由页面（做 redirect 或删除）

## 实现文件
- `components/layout/Sidebar.tsx` — 最终精简到 6 个入口
- `app/agent-team/page.tsx` — 新增 Tools & Skills Tab
- `app/capabilities/page.tsx` — 改为 redirect 到 `/agent-team`
- `app/docs/page.tsx` — 保留但从侧边栏移除
- `app/messages/page.tsx` — 整合到任务详情
- `app/monitor/page.tsx` — 删除或 redirect 到设置
- `app/tags/page.tsx` — 整合到任务管理筛选

## 依赖关系
- 前置：38、39、40、44（先完成各个合并任务）
- 后续：42【P1】页面宽度与样式全局统一

## 验证方式
1. 侧边栏精确包含 6 个入口：项目导入、任务管理、版本管理、Token管理、成员管理、Agent团队 + 底部设置
2. 所有被合并/移除的功能通过其他入口仍可访问
3. 无死链或 404
4. Agent 团队页的 Tools & Skills Tab 功能完整
5. **至少 5 轮修改验证**：侧边栏→Agent团队→redirect→死链检查→整体体验
