# 43【P1】版本管理页重构 - 版本时间线

## 背景
版本管理的展示和组织方式有问题，应该是一个高效简约的页面。核心需求：展示从最初版本到最新版本的完整时间线，每个版本的备注、信息、代码产物下载、时间等。

## 目标
- 重新设计版本管理页为**版本时间线**视图
- 每个版本节点展示：版本号、发布时间、备注/变更说明、代码产物下载链接
- 时间线从最新到最旧，支持按时间范围筛选
- 简洁高效，一眼看到全貌

## 技术方案
1. 设计垂直时间线布局（类似 GitHub Release 页面风格）
2. 每个版本节点包含：
   - 版本号（大字号）
   - 发布时间
   - 变更说明/备注（Markdown 渲染）
   - 下载按钮（代码产物 zip/tar.gz）
   - 关联的分支信息
3. 顶部操作栏：创建新版本、筛选
4. 分支管理作为页面内的次级 Tab（合并自 #38）

## 实现文件
- `app/versions/page.tsx` — 重构为时间线视图
- `components/versions/VersionTimeline.tsx` — 新时间线组件
- `components/versions/VersionCard.tsx` — 单个版本节点组件

## 验证方式
1. 版本从新到旧按时间线排列，信息完整
2. 每个版本的下载链接可用
3. 页面加载速度快（版本多时支持分页/虚拟滚动）
4. 移动端布局合理
5. **至少 5 轮修改验证**：数据展示→下载功能→排序筛选→响应式→暗色模式
