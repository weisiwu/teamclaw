# 【P1】M9 打包机制检测服务

> 优先级：P1（中）
> 前置依赖：H5（导入流程异步编排串联）完成后接入
> 关联模块：[项目导入模块](../modules/项目导入模块.md)

---

## 1. 问题描述

项目导入模块文档定义的 11 步流程中，**第 5 步「了解打包/编译机制」** 是唯一没有对应服务文件的步骤。

当前已有的 `techDetector.ts` 能识别 `buildTool`（npm/cargo/go 等包管理器），但不分析具体的打包/编译配置：

| 已有能力 | 缺失能力 |
|---------|---------|
| 识别使用了 npm/yarn/pnpm | 分析 webpack/vite/rollup/turbopack 配置 |
| 识别 package.json 存在 | 解析 scripts 中的 build/dev/start 命令 |
| 识别 Cargo.toml 存在 | 分析构建产物路径和格式 |

---

## 2. 目标

创建 `server/src/services/buildDetector.ts`，实现以下功能：

### 2.1 检测打包工具

从配置文件中识别实际使用的打包工具：

| 配置文件 | 打包工具 |
|---------|---------|
| `vite.config.ts/js` | Vite |
| `webpack.config.ts/js` | Webpack |
| `rollup.config.ts/js` | Rollup |
| `next.config.ts/js/mjs` | Next.js (内置 webpack/turbopack) |
| `nuxt.config.ts` | Nuxt (内置 vite/webpack) |
| `tsconfig.json` + 无 bundler | tsc (TypeScript Compiler) |
| `esbuild` 相关配置 | esbuild |
| `Cargo.toml` | cargo build |
| `go.mod` | go build |
| `Makefile` / `CMakeLists.txt` | make / cmake |

### 2.2 解析构建脚本

从 `package.json` 的 `scripts` 字段提取：

```typescript
interface BuildScripts {
  dev?: string;       // 开发启动命令
  build?: string;     // 生产构建命令
  start?: string;     // 生产启动命令
  test?: string;      // 测试命令
  lint?: string;      // 代码检查命令
  typeCheck?: string;  // 类型检查命令
}
```

### 2.3 分析构建产物

```typescript
interface BuildOutput {
  outputDir: string;      // 构建输出目录（dist/build/.next/out 等）
  format?: string;        // 产物格式（esm/cjs/umd）
  hasSourceMap?: boolean; // 是否生成 source map
}
```

---

## 3. 输出接口

```typescript
export interface BuildMechanism {
  bundler: string;              // 主打包工具名称
  bundlerVersion?: string;      // 版本号（从 package.json 提取）
  configFile?: string;          // 配置文件路径
  scripts: BuildScripts;        // 构建相关脚本
  output: BuildOutput;          // 产物信息
  features: string[];           // 特性列表（如 "HMR", "SSR", "Code Splitting"）
  monorepo?: boolean;           // 是否 monorepo（检测 workspaces/lerna/turborepo）
  monorepoTool?: string;        // monorepo 工具名称
}
```

## 4. 实施步骤

| # | 操作 | 说明 |

| 4 | 分析构建产物 | 读取打包工具配置文件中的 output 配置 |
| 5 | 检测 monorepo | 检查 workspaces/lerna.json/turbo.json/pnpm-workspace.yaml |
| 6 | 接入 importOrchestrator | 作为第 5 步执行 |
