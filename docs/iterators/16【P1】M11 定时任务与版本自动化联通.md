# 【P1】M11 定时任务与版本自动化联通

> 优先级：P1（中）
> 前置依赖：H6（Agent LLM 执行）、H7（流水线串联）
> 关联模块：定时任务管理、版本机制、辅助能力、任务机制

---

## 1. 问题描述

架构文档涉及多处「模块联动」需求，当前各模块独立运行但缺少实际联通：

| 联动需求 | 上游模块 | 下游模块 | 现状 |
|---------|---------|---------|------|
| 定时任务执行 prompt | `cronService.trigger()` | 群聊发送 / Agent 执行 | ❌ 模拟执行（90% 随机成功） |
| 任务完成 → 自动版本升级 | `taskLifecycle.transition('done')` | `autoBump.bump()` | ❌ autoBump 存在但未注册为 hook |
| 聊天文件 → 文档库 | `POST /messages/file` | `docService.create()` | ❌ 文件消息只入队列，不存入文档库 |
| 任务记忆 → 向量化 | `taskMemory` | `vectorStore` | ❌ 任务记忆仅存内存，不做向量化 |
| 聊天记录 → 向量化 | `messageQueue` | `vectorStore` | ❌ 聊天历史仅存内存数组 |

---

## 2. 改造目标

### 2.1 定时任务真实执行

```
cronService.trigger(jobId)
  ├── 获取 job.prompt
  ├── 调用 messagePipeline（或直接 llmCall）
  │     ├── 如果 prompt 含 @agent → 走 Agent 执行流程
  │     └── 否则 → 直接 LLM 调用 → 发送结果到群聊
  ├── 记录执行结果（非随机模拟）
  └── 更新 job 统计
```

### 2.2 任务完成 → autoBump

```typescript
// taskLifecycleHooks.ts
taskLifecycle.onStatusChange(async (task, oldStatus, newStatus) => {
  if (newStatus === 'done') {
    // 触发自动版本升级
    const bumpResult = await autoBump.bump(task.projectId, 'patch', {
      reason: `任务 ${task.taskId} 完成`,
      commitMessage: task.title,
    });
    // 群聊通知
    await messageReply.send(task.channel, `任务「${task.title}」已完成，版本升级至 ${bumpResult.newVersion}`);
  }
});
```

### 2.3 聊天文件 → 文档库

```
POST /messages/file
  ├── 消息入队列 ✅（已有）
  ├── 【新增】调用 docService.create() 存入文档库
  ├── 【新增】如果是可解析类型（PDF/Word/Excel/Markdown）→ docParser 解析
  └── 【新增】解析结果向量化存储
```

### 2.4 记忆向量化

```
任务完成时：
  taskMemory.getContext(taskId) → 提取摘要 → vectorStore.add()

消息完成处理时：
  messageQueue.onCompleted → 消息内容 → vectorStore.add()
```

## 3. 实施步骤

| # | 操作 | 涉及文件 |
|---|------|---------|
| 1 | `cronService.trigger()` 替换模拟逻辑为真实 LLM/消息管道调用 | `server/src/services/cronService.ts` |

## 5. 验收标准

| # | 验收项 | 验证方式 |
|---|--------|---------|
| 1 | 定时任务触发后 LLM 实际执行 | 检查 token 消耗、执行日志 |
| 2 | 任务完成后 git tag 自动创建 | 检查 git log --tags |
| 3 | 上传文件消息后文档库出现对应条目 | GET /docs 列表验证 |
| 4 | 可解析文件（PDF/Markdown）内容被提取 | 检查文档内容字段 |
| 5 | 任务完成时记忆摘要写入向量库 | 查询 ChromaDB 验证 |
| 6 | 消息处理完成后内容写入向量库 | 查询 ChromaDB 验证 |

## 6. 架构文档对应关系

| 架构文档要求 | 对应实现 |
|------------|---------|
| 「定时任务只支持 prompt，口头描述后创建 prompt」 | cronService → 真实 LLM 执行 |
| 「任务完成后，自动版本升级」 | taskLifecycleHooks → autoBump |
| 「群聊文档消息都会被上传，生成文档」 | POST /messages/file → docService |
| 「任务需要有长期记忆，保存在向量数据库中」 | taskMemory → vectorStore |
| 「群聊回话记录做保存，并且向量化」 | messageQueue → vectorStore |
| 「项目需要有长期代码记忆」 | 结合 H5 导入流程的向量化步骤 |
