# TeamClaw 迭代任务管理

> 本目录是 TeamClaw 项目的任务生命周期管理中心。

## 目录结构

```
iterators/
├── README.md          # 本索引文件
├── processing/        # 待执行 & 进行中的任务（按执行优先级编号）
│   └── README.md      # 任务总览 & 执行顺序说明
└── done/              # 已完成的任务归档
    └── README.md      # 完成任务索引
```

## 任务生命周期

```
新需求 / Bug  →  processing/（编号排序）  →  done/（完成归档）
```

- **processing/** — 所有待执行和进行中的任务，按**执行优先级**编号（01 = 最优先）
- **done/** — 所有已完成的任务归档，保留完整记录

## 命名规范

```
{序号}【{优先级}】{任务名称}.md
```

- **序号**：两位数字，表示执行优先级（01 最高）
- **优先级标签**：`P0` 阻塞级 > `P1` 重要 > `P2` 改进 > `P3` 锦上添花
- **任务名称**：简洁描述，中文优先

示例：`05【P0】API Token 数据模型与 CRUD 接口.md`

## 注意事项

1. **每个项目独立管理** — poetry-app 的迭代任务在 `apps/poetry-app/docs/iterators/`
2. **任务完成后**移入 `done/`，保留原始编号（在 done/ 内重新编号）
3. **优先级可调整** — 根据实际情况更新序号和优先级标签
4. **关联 issues/** — 每轮迭代结束后在 `issues/` 生成回顾总结
