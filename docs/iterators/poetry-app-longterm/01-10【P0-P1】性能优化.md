# poetry-app 性能优化迭代（01-10轮）

> 本阶段聚焦性能优化，覆盖首屏加载、列表渲染、API缓存、搜索性能等方面。

---

## 01【P0】首屏加载优化
- 分析首屏加载瓶颈
- 优化关键渲染路径
- 添加 loading skeleton
- 目标：FCP < 1.5s

## 02【P0】列表虚拟滚动
- 诗词列表虚拟滚动实现
- 诗人列表虚拟滚动
- 目标：万级数据流畅滚动

## 03【P0】图片优化
- 图片懒加载实现
- WebP/AVIF 格式支持
- 响应式图片
- 目标：减少 50% 图片流量

## 04【P1】API 缓存策略
- React Query 配置优化
- SWR vs React Query 选择
- 缓存失效策略
- 目标：减少重复请求

## 05【P1】搜索性能优化
- 搜索防抖优化
- 搜索结果分页
- 热门搜索缓存
- 目标：搜索响应 < 200ms

## 06【P1】Bundle 优化
- Code Splitting 检查
- Tree Shaking 验证
- 第三方库按需加载
- 目标：首屏 JS < 150KB

## 07【P1】ISR 配置优化
- 检查 ISR revalidate 时间
- 按页面分类配置
- 缓存策略细化
- 目标：CDN 命中率 > 80%

## 08【P1】数据库查询优化
- 检查 Supabase 查询计划
- 添加必要索引
- 优化 N+1 问题
- 目标：API 响应 < 300ms

## 09【P2】性能监控
- Web Vitals 集成
- 性能指标上报
- 性能仪表盘
- 目标：建立性能基线

## 10【P2】性能回归测试
- Lighthouse CI 集成
- 性能预算设置
- 自动化性能测试
- 目标：防止性能退化
