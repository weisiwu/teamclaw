## 2. 测试策略

### 2.1 测试层次

```
                    ┌──────────────┐
                    │  E2E 测试     │  ← 暂不做（需要完整环境）
                    ├──────────────┤
                    │  API 集成测试 │  ← 本任务重点
                    ├──────────────┤
                    │  单元测试     │  ← 已有部分覆盖
                    └──────────────┘
```

### 2.2 测试工具选型

| 工具 | 用途 |
|------|------|
| Vitest | 测试框架（项目已安装） |
| supertest | HTTP 端点测试（需安装） |
| 内存 SQLite / mock | 数据层隔离 |

### 2.3 测试原则

- 每个路由文件对应一个测试文件
- 测试 **HTTP 层**（状态码、响应格式、边界条件）
- 使用 mock 隔离外部依赖（DB、Redis、LLM）
- 测试文件命名：`tests/routes/xxx.test.ts`

## 4. 测试基础设施

### 4.1 安装依赖

```bash
cd server
npm install -D supertest @types/supertest
```

### 4.2 创建测试工具

**新建 `tests/helpers/setup.ts`**：

```typescript
import express from 'express';
import { unifiedErrorHandler, notFoundHandler } from '../../server/src/middleware/errorHandler';

/**
 * 创建测试用 Express app
 * 挂载指定路由 + 错误处理
 */
export function createTestApp(routePath: string, router: express.Router) {
  const app = express();
  app.use(express.json());
  app.use(routePath, router);
  app.use(notFoundHandler);
  app.use(unifiedErrorHandler);
  return app;
}
```

**新建 `tests/helpers/auth.ts`**：

```typescript
/**
 * 生成测试用认证 header
 */
export function authHeaders(role: string = 'admin', userId: string = 'test_user') {
  return {
    'X-User-Id': userId,
    'X-User-Role': role,
  };
  // H2 完成后改为：返回 Bearer Token
}
```
