# 【P1】M5 连接优雅关闭

> 优先级：P1（中）
> 前置依赖：无 · 后续影响：服务部署与运维可靠性

---

## 1. 问题描述

服务进程退出时（部署更新、手动重启、异常退出），没有关闭数据库连接和 Redis 连接：

| 连接 | 文件 | 当前状态 |
|------|------|---------|
| PostgreSQL 连接池 | `server/src/utils/db.ts` | `pool.on('error', ...)` 仅记录错误，无关闭逻辑 |
| Redis 连接 | `server/src/utils/redis.ts` | `redis.on('error', ...)` 仅记录错误，无关闭逻辑 |
| SQLite 连接 | `server/src/db/sqlite.ts` | 有 `closeDb()` 函数但未被调用 |
| Express HTTP Server | `server/src/index.ts` | `app.listen()` 返回值未保存，无法 `server.close()` |

### 风险

| 场景 | 后果 |
|------|------|
| Docker 容器重启 | PostgreSQL 连接未正常释放，连接池耗尽 |
| `kill -15 <pid>` 停服务 | 正在处理的请求被强制中断，可能导致数据不一致 |
| Redis 未断开 | 连接泄漏，Redis `maxclients` 可能被耗尽 |
| 新部署时旧进程未完全退出 | 端口被占用，新进程启动失败 |

## 3. 修复方案

### 3.1 创建统一的 shutdown 管理器

**新建 `server/src/utils/shutdown.ts`**：

```typescript
type ShutdownHandler = () => Promise<void> | void;

const handlers: Array<{ name: string; fn: ShutdownHandler }> = [];
let isShuttingDown = false;

/**
 * 注册关闭处理函数（LIFO 顺序执行，先注册的后执行）
 */
export function onShutdown(name: string, fn: ShutdownHandler): void {
  handlers.push({ name, fn });
}

/**
 * 执行所有关闭处理
 */
async function gracefulShutdown(signal: string): Promise<void> {
  if (isShuttingDown) return;
  isShuttingDown = true;

  console.log(`\n[shutdown] Received ${signal}, graceful shutdown starting...`);

  // LIFO 顺序：先关应用层，再关基础设施
  const reversed = [...handlers].reverse();

  for (const { name, fn } of reversed) {
    try {
      console.log(`[shutdown] Closing ${name}...`);
      await Promise.race([
        Promise.resolve(fn()),
        new Promise((_, reject) =>
          setTimeout(() => reject(new Error('timeout')), 5000)
        ),
      ]);
      console.log(`[shutdown] ${name} closed`);
    } catch (err) {
      console.error(`[shutdown] ${name} close error:`, err);
    }
  }

  console.log('[shutdown] All connections closed, exiting');
  process.exit(0);
}

/**
 * 注册进程信号监听（在 index.ts 中调用一次）
 */
export function registerShutdownHandlers(): void {
  process.on('SIGTERM', () => gracefulShutdown('SIGTERM'));
  process.on('SIGINT', () => gracefulShutdown('SIGINT'));

  // 未捕获异常也尝试优雅关闭
  process.on('uncaughtException', (err) => {
    console.error('[shutdown] Uncaught exception:', err);
    gracefulShutdown('uncaughtException');
  });

  process.on('unhandledRejection', (reason) => {
    console.error('[shutdown] Unhandled rejection:', reason);
    // 不退出，仅记录
  });
}
```

### 3.2 修改 db.ts

```typescript
// server/src/utils/db.ts
import pg from 'pg';
import { onShutdown } from './shutdown.js';

const { Pool } = pg;

export const pool = new Pool({
  connectionString: process.env.DATABASE_URL || 'postgresql://localhost:5432/teamclaw',
  max: 20,
  idleTimeoutMillis: 30000,
  connectionTimeoutMillis: 2000,
});

pool.on('error', (err) => {
  console.error('Unexpected error on idle client', err);
});

// 注册优雅关闭
onShutdown('PostgreSQL', async () => {
  await pool.end();
});
```

### 3.4 修改 index.ts

```typescript
// server/src/index.ts
import { onShutdown, registerShutdownHandlers } from './utils/shutdown.js';

// 注册信号处理（文件顶部，其他 import 之后）
registerShutdownHandlers();

// ... 中间件、路由注册 ...

// 保存 server 引用
const server = app.listen(PORT, () => {
  console.log(`Server running on http://localhost:${PORT}`);
  registerAutoBumpHook();
});

// 注册 HTTP Server 关闭
onShutdown('HTTP Server', async () => {
  return new Promise<void>((resolve, reject) => {
    // 停止接受新连接，等待现有请求完成
    server.close((err) => {
      if (err) reject(err);
      else resolve();
    });
  });
});
```

## 4. 关闭顺序

注册顺序与执行顺序（LIFO）：

```
注册顺序（代码加载）：
  1. PostgreSQL (db.ts)
  2. Redis (redis.ts)
  3. SQLite (sqlite.ts)
  4. HTTP Server (index.ts)

关闭顺序（LIFO）：
  1. HTTP Server  — 停止接受新请求，等待现有请求完成
  2. SQLite       — 关闭 SQLite 连接
  3. Redis        — 断开 Redis 连接
  4. PostgreSQL   — 关闭连接池（最后关，因为其他服务可能依赖它）
```

## 6. 验收标准

| # | 验收项 | 验证方式 |
|---|--------|---------|
| 1 | `Ctrl+C` 停止服务时输出关闭日志 | 终端观察 `[shutdown] Closing xxx...` |
| 2 | `kill -15 <pid>` 触发优雅关闭 | 发送信号后观察日志 |
| 3 | 关闭后进程退出码为 0 | `echo $?` 检查 |
| 4 | PostgreSQL 连接全部释放 | `SELECT count(*) FROM pg_stat_activity WHERE application_name = 'teamclaw'` → 0 |
| 5 | 关闭期间的正在处理请求正常完成 | 发送长请求 + 立即 kill，验证响应完整 |
| 6 | 单个 handler 超时（5s）不阻塞其他 | mock 一个永远不 resolve 的 handler |
