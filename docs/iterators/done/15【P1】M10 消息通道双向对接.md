# 【P1】M10 消息通道双向对接（飞书发送 + 微信接入）

> 优先级：P1（中）
> 前置依赖：H7（消息→任务→Agent 流水线串联）
> 关联模块：消息机制、流程

---

## 1. 问题描述

架构文档定义的流程入口：

```
workbuddy: 接入微信消息
enhanced gateway → channel/tui/browser → openclaw gateway
```

代码中已实现消息的「接收」适配，但缺少「发送」能力和微信实际接入：

| 通道 | 接收（入站） | 发送（出站） |
|------|------------|------------|
| 飞书 | ✅ `feishu.ts` 路由接收 webhook + `feishuService.ts` API 封装 | ❌ 无发送消息 API（无 `sendFeishuMessage()`） |
| 微信 | ⚠️ `channelAdapter.ts` 有格式适配器 | ❌ 无微信 Bot SDK 接入、无 webhook 路由 |
| Web | ✅ `adaptWebMessage()` 格式转换 | ⚠️ 只能通过前端轮询 API，无 WebSocket 推送 |

**后果：**
- Agent 执行完成后无法回复飞书群聊
- 微信消息无法进入系统
- 用户无法通过聊天平台与系统交互（只能用 Web 后台）

---

## 2. 改造目标

### 2.1 飞书出站：发送消息

```typescript
// 新增 feishuService.ts 发送能力
async function sendFeishuMessage(params: {
  appId: string;
  appSecret: string;
  receiveIdType: 'chat_id' | 'open_id';
  receiveId: string;
  msgType: 'text' | 'interactive' | 'post';
  content: string;
}): Promise<{ messageId: string }>
```

使用场景：
- Agent 执行完成 → 回复群聊
- 任务状态变更 → 群聊通知
- 抢占发生 → 通知被暂停的用户
- 定时任务触发 → 发送 prompt 到群聊

### 2.2 微信接入

采用 `wechaty` 或类似 SDK 接入微信群聊：

```
微信群消息 → wechaty → POST /api/v1/messages (channel=wechat) → 消息管道
Agent 回复 → wechaty.say() → 微信群
```

### 2.3 Web WebSocket 推送（可选，后续优化）

当前 Web 端通过 REST 轮询获取消息。可改为 WebSocket 推送实现实时通信。
