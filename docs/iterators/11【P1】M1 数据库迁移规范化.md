> 优先级：P1（中）
> 前置依赖：H1（数据存储统一到 PostgreSQL） · 后续影响：所有涉及 Schema 变更的开发

---

## 1. 问题描述

`server/src/db/migrations/run.ts`（225 行）中使用以下模式做列添加：

```typescript
try {
  db.prepare("ALTER TABLE versions ADD COLUMN git_tag TEXT").run();
} catch (e: unknown) {
  // Column may already exist, ignore
}
```

这种 try-catch 重复了 **10+ 次**，存在以下问题：

| 问题 | 说明 |
|------|------|
| 无版本追踪 | 无法知道当前数据库执行到了哪个迁移 |
| 不可审计 | 没有执行时间、执行者的记录 |
| 静默吞错 | 不仅 "column exists" 被忽略，其他异常也被吞掉 |
| 不可回滚 | 没有 down 迁移，无法回退 Schema 变更 |
| 命名混乱 | 迁移文件有 `.ts` 和 `.sql` 两种格式，命名前缀不统一（`004_` vs `20260319_`） |

## 2. 目标状态

- **迁移版本表**：数据库中有 `_migrations` 表记录已执行的迁移
- **幂等执行**：`runMigrations()` 只执行未记录的迁移，已执行的自动跳过
- **统一格式**：所有迁移文件使用 `YYYYMMDD_NNN_description.sql` 命名
- **支持回滚**：每个迁移有 up/down 两部分（down 可选）
- **执行日志**：记录每个迁移的执行时间、耗时、状态

## 3. 实现步骤

### Step 1：创建迁移管理表（0.5h）

```sql
-- 迁移记录表（在 PostgreSQL 中创建）
CREATE TABLE IF NOT EXISTS _migrations (
  id SERIAL PRIMARY KEY,
  name TEXT UNIQUE NOT NULL,          -- 迁移文件名，如 '20260319_001_add_search_history'
  executed_at TIMESTAMPTZ DEFAULT NOW(),
  checksum TEXT,                       -- 文件内容 hash，检测篡改
  execution_time_ms INTEGER,           -- 执行耗时
  status TEXT DEFAULT 'success'        -- success | failed
);
```

### Step 2：重写迁移运行器（2h）

**重写 `server/src/db/migrations/run.ts`**：

```typescript
import { pool } from '../../utils/db.js';
import fs from 'fs';
import path from 'path';
import crypto from 'crypto';

const MIGRATIONS_DIR = path.join(import.meta.dirname, '.');

interface MigrationFile {
  name: string;
  path: string;
  checksum: string;
}

// 确保迁移表存在
async function ensureMigrationTable(): Promise<void> {
  await pool.query(`
    CREATE TABLE IF NOT EXISTS _migrations (
      id SERIAL PRIMARY KEY,
      name TEXT UNIQUE NOT NULL,
      executed_at TIMESTAMPTZ DEFAULT NOW(),
      checksum TEXT,
      execution_time_ms INTEGER,
      status TEXT DEFAULT 'success'
    )
  `);
}

// 获取已执行的迁移
async function getExecutedMigrations(): Promise<Set<string>> {
  const { rows } = await pool.query(
    'SELECT name FROM _migrations WHERE status = $1 ORDER BY name',
    ['success']
  );
  return new Set(rows.map(r => r.name));
}

// 扫描迁移文件
function scanMigrationFiles(): MigrationFile[] {
  return fs.readdirSync(MIGRATIONS_DIR)
    .filter(f => f.endsWith('.sql') && /^\d{8}_/.test(f))
    .sort()
    .map(f => {
      const content = fs.readFileSync(path.join(MIGRATIONS_DIR, f), 'utf-8');
      return {
        name: f.replace('.sql', ''),
        path: path.join(MIGRATIONS_DIR, f),
        checksum: crypto.createHash('md5').update(content).digest('hex'),
      };
    });
}

// 执行单个迁移
async function executeMigration(migration: MigrationFile): Promise<void> {
  const sql = fs.readFileSync(migration.path, 'utf-8');
  const upSection = sql.split('-- DOWN')[0]; // 只执行 UP 部分

  const start = Date.now();
  const client = await pool.connect();
  try {
    await client.query('BEGIN');
    await client.query(upSection);
    await client.query(
      'INSERT INTO _migrations (name, checksum, execution_time_ms, status) VALUES ($1, $2, $3, $4)',
      [migration.name, migration.checksum, Date.now() - start, 'success']
    );
    await client.query('COMMIT');
    console.log(`  ✅ ${migration.name} (${Date.now() - start}ms)`);
  } catch (err) {
    await client.query('ROLLBACK');
    await pool.query(
      'INSERT INTO _migrations (name, checksum, execution_time_ms, status) VALUES ($1, $2, $3, $4) ON CONFLICT (name) DO NOTHING',
      [migration.name, migration.checksum, Date.now() - start, 'failed']
    );
    console.error(`  ❌ ${migration.name}: ${err}`);
    throw err;
  } finally {
    client.release();
  }
}

// 主入口
export async function runMigrations(): Promise<void> {
  console.log('[migrations] Checking database migrations...');
  await ensureMigrationTable();

  const executed = await getExecutedMigrations();
  const files = scanMigrationFiles();
  const pending = files.filter(f => !executed.has(f.name));

  if (pending.length === 0) {
    console.log('[migrations] Database is up to date');
    return;
  }

  console.log(`[migrations] Running ${pending.length} pending migration(s)...`);
  for (const migration of pending) {
    await executeMigration(migration);
  }
  console.log('[migrations] All migrations completed');
}
```

### Step 4：迁移文件格式规范（0.5h）

每个 `.sql` 文件遵循格式：

```sql
-- UP
CREATE TABLE IF NOT EXISTS xxx ( ... );
CREATE INDEX IF NOT EXISTS idx_xxx ON xxx(...);

-- DOWN
DROP TABLE IF EXISTS xxx;
```

async function bootstrap() {
  await runMigrations();
  // ... 其余启动逻辑
  app.listen(PORT, () => { ... });
}
bootstrap();

## 5. 验收标准

| # | 验收项 | 验证方式 |
|---|--------|---------|
| 1 | `_migrations` 表存在且有记录 | `SELECT * FROM _migrations` |
| 2 | 重复执行 `runMigrations()` 不报错 | 启动两次服务 |
| 3 | 所有迁移文件使用统一命名格式 | `ls migrations/*.sql` |
| 4 | 新增迁移只需新建 `.sql` 文件 | 创建测试迁移验证 |
| 5 | 失败的迁移在表中标记为 `failed` | 构造一个错误 SQL 测试 |
| 6 | 无重复的迁移文件 | 文件名去重检查 |
